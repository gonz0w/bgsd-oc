<objective>
Establish one shared debug or verbosity contract for touched CLI and plugin reliability diagnostics so investigation output is predictable and normal runs stay quiet.

Purpose: The audit found logging drift across `BGSD_DEBUG`, `GSD_DEBUG`, and `NODE_ENV=development`, which makes reliability diagnostics hard to reason about and leaves plugin-only hot paths with different debug behavior than the CLI. This plan creates the shared contract first so later noise-reduction work can remove chatter without fragmenting operator escape hatches.
Output: Shared debug-gating helpers, migrated touched plugin or CLI diagnostics, rebuilt runtime bundle, and focused regressions that prove the contract is consistent across the touched flows.
</objective>

> **TDD Decision:** Selected — the logging contract is deterministic about which markers appear under default versus explicit debug settings, so regression tests should lock that behavior before implementation.
