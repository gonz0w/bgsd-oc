**In src/commands/lessons.js — add `cmdLessonsAnalyze(cwd, options, raw)`:**

1. Read all entries from `.planning/memory/lessons.json`
2. Group lessons by each affected agent (a lesson with `affected_agents: ['bgsd-executor', 'bgsd-planner']` counts in both groups)
3. Within each agent group, sub-group by `type` to identify recurrent patterns
4. Filter to only show groups with ≥2 supporting lessons (per LESSON-04 requirement)
5. For each qualifying group, compute:
   - `agent`: the agent name
   - `pattern_type`: the lesson type (e.g., 'agent-behavior')
   - `count`: number of lessons in this group
   - `severity_distribution`: count of each severity level
   - `common_root_causes`: deduplicated root causes from the group
   - `lessons`: array of { id, title, date, severity } for each contributing lesson
6. Sort groups by count descending (most recurrent first)
7. If `--agent` option provided, filter to that agent only
8. Output the grouped patterns with summary stats

**In src/commands/lessons.js — add `cmdLessonsSuggest(cwd, options, raw)`:**

1. Read all entries from `.planning/memory/lessons.json`
2. Filter out `type: 'environment'` entries (migrated free-form lessons produce 0 suggestions — LESSON-02 requirement)
3. Group remaining by affected agent + type (same grouping as analyze)
4. Filter to groups with ≥2 lessons (threshold per LESSON-05)
5. For each qualifying group, generate a structured suggestion:
   - `agent`: target agent name
   - `suggestion_type`: 'behavioral' (for agent-behavior) | 'workflow' (for workflow) | 'tooling' (for tooling)
   - `summary`: Synthesize from the group's common root causes — "Agent {X} repeatedly encounters {root_cause_pattern}. Consider: {prevention_rules}"
   - `supporting_lessons`: count and IDs of lessons backing this suggestion
   - `severity`: highest severity in the group (CRITICAL > HIGH > MEDIUM > LOW)
   - `prevention_rules`: deduplicated prevention rules from the group
6. If `--agent` option provided, filter suggestions to that agent only
7. Sort by severity descending, then by supporting lesson count
8. Output suggestions with advisory note: "These are suggestions only — review and apply manually"

Export both new functions from module.exports.

  <action>
**In src/commands/lessons.js — add `cmdLessonsCompact(cwd, options, raw)`:**

1. Read all entries from `.planning/memory/lessons.json`
2. Check entry count against threshold (default 100, configurable via `--threshold` option)
3. If below threshold: output `{ compacted: false, reason: 'Below threshold', count, threshold }` and return
4. Group entries by normalized root_cause (lowercase, trimmed) — entries with identical root causes are duplicates
5. For each group with >1 entry:
   - Keep only the entry with the latest date
   - Preserve any unique prevention_rules across the group (merge into the kept entry's prevention_rule if different)
   - Preserve the highest severity from the group
   - Add `compacted_from: N` field to the kept entry (where N = original group size)
6. Write the deduplicated array back to `.planning/memory/lessons.json`
7. Clear and rewrite SQLite entries if available (try/catch, non-blocking)
8. Output: `{ compacted: true, before: originalCount, after: newCount, removed: diff, groups_merged: N }`

Export `cmdLessonsCompact` from module.exports.
