  <files>src/commands/lessons.js</files>
  <action>
Create a new file `src/commands/lessons.js` with the following:

**Imports and Constants:**
- Require `fs`, `path`, `crypto` from Node builtins
- Require `{ output, error, debugLog }` from `../lib/output`
- Define `LESSON_SCHEMA` object describing required fields:
  ```
  LESSON_SCHEMA = {
    required: ['title', 'severity', 'type', 'root_cause', 'prevention_rule', 'affected_agents'],
    severity_values: ['LOW', 'MEDIUM', 'HIGH', 'CRITICAL'],
    type_values: ['workflow', 'agent-behavior', 'tooling', 'environment']
  }
  ```

**`validateLesson(entry)`:**
- Check each required field exists and is non-empty string (affected_agents can also be an array)
- Validate severity is one of LESSON_SCHEMA.severity_values (case-insensitive, normalize to uppercase)
- Validate type is one of LESSON_SCHEMA.type_values (case-insensitive, normalize to lowercase)
- Return `{ valid: true, normalized }` with normalized entry (severity uppercased, type lowercased, affected_agents always array split by comma if string)
- Return `{ valid: false, errors: ['field X is required', ...] }` on failure

**`cmdLessonsCapture(cwd, options, raw)`:**
- Extract from options: `title`, `severity`, `type`, `rootCause` (--root-cause), `prevention` (--prevention), `agents` (--agents)
- Build entry object with fields: `id` (crypto.randomUUID()), `date` (new Date().toISOString()), `title`, `severity`, `type`, `root_cause`, `prevention_rule`, `affected_agents`
- Call `validateLesson(entry)` — if invalid, call `error()` with all validation errors
- Read existing lessons from `.planning/memory/lessons.json` (create dir + empty array if not exists)
- Append validated entry, write back to JSON
- Dual-write to SQLite via PlanningCache if available (try/catch, non-blocking)
- Output success with entry id and summary

**`cmdLessonsList(cwd, options, raw)`:**
- Extract from options: `type`, `severity`, `since`, `limit`, `query`
- Read all entries from `.planning/memory/lessons.json`
- Filter by type (case-insensitive match) if --type provided
- Filter by severity (case-insensitive match) if --severity provided
- Filter by date >= since if --since provided (ISO date string comparison)
- Filter by query (case-insensitive substring match on title, root_cause, prevention_rule) if --query provided
- Sort by date descending (newest first)
- Apply limit (default 20 if not specified)
- Output entries with count, total, and applied filters

Export all functions: `validateLesson`, `cmdLessonsCapture`, `cmdLessonsList`, `LESSON_SCHEMA`.
