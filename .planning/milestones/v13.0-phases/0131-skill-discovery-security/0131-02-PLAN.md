Add to `src/commands/skills.js`:

**`parseGitHubUrl(source)` helper:**
- Accept formats: `https://github.com/owner/repo`, `github.com/owner/repo`, `owner/repo`
- Return `{ owner, repo }` or `null` if invalid
- Repo root only — no subdirectory support (per CONTEXT.md)

**`async fetchGitHubContents(owner, repo)` function:**
- Use Node.js built-in `fetch()` (zero dependencies, following misc.js pattern)
- Fetch `https://api.github.com/repos/${owner}/${repo}/contents` with `Accept: application/vnd.github.v3+json`
- Handle rate limiting (403) with clear error message
- Handle 404 with "Repository not found" error
- Return array of `{ name, path, size, download_url, type }` entries
- For directories: recursively fetch contents (skills may have subdirs for resources)
- Filter to files only for download

**`async downloadFiles(files)` function:**
- For each file with `download_url`: fetch raw content
- Return array of `{ path, content, size }`
- Handle fetch failures per-file with clear error

**`cmdSkillsInstall(cwd, options, raw)` — async function:**

Flow:
1. Parse `options.source` with `parseGitHubUrl()` — error if invalid
2. Extract skill name from repo name
3. Check if `.agents/skills/${name}` already exists — error with "Skill '${name}' already installed. Run skills:remove first."
4. Fetch repo contents via GitHub API
5. Verify SKILL.md exists in file list — error with "Invalid skill: no SKILL.md found in repository root" if missing
6. Download all files
7. Write files to a temp directory (e.g., `os.tmpdir()`)
8. Run `scanSkillFiles(tempDir)` on downloaded content
9. Format and display scan results using `formatScanResults()`
10. **If verdict is `'dangerous'`:** Log audit entry with `outcome: 'blocked'`, display "Install BLOCKED", clean up temp dir, exit
11. Show file list + total size: `"Install skill '${name}' (${fileCount} files, ${totalSize})?"` 
12. **If verdict is `'warn'`:** Append warning count to confirmation prompt: `"(${warnCount} warnings)"`
13. Output confirmation prompt — the agent (executor) handles Y/N interaction
14. **On confirmation:** Create `.agents/skills/${name}/` directory, write all files, log audit entry with `outcome: 'installed'`
15. **On rejection:** Log audit entry with `outcome: 'rejected'`, clean up temp dir
16. Success message: one-liner with path: `"Installed '${name}' to .agents/skills/${name}/ (${fileCount} files)"`

**Important:** The install command outputs the confirmation prompt and scan results. The calling agent/workflow handles the actual Y/N interaction. The command itself is structured as: fetch → scan → output results → return `{ requires_confirmation: true, ... }` in JSON mode, or print prompt in TTY mode. A separate `cmdSkillsConfirmInstall(cwd, options, raw)` function completes the write if confirmed.

Actually, simpler approach following CLI patterns: `cmdSkillsInstall` does fetch + scan + display. If clean or warn, it outputs `{ action: 'confirm', name, files, scan }`. A `--confirm` flag on the same command triggers the actual file write. This avoids needing a second command.

Flow refined:
- Without `--confirm`: fetch, scan, show results, output confirmation data
- With `--confirm --source <url>`: re-fetch (or use cached temp), write files, log audit
- Blocked installs (dangerous): never reach confirmation stage, audit logged immediately
