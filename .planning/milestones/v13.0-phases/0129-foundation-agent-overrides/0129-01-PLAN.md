<task type="auto">
  <name>Task 1: Add YAML validation, content sanitization, and unified diff utilities</name>
  <files>src/commands/agent.js</files>
  <action>
Add three utility functions to src/commands/agent.js (above the existing command functions):

**`validateAgentFrontmatter(content)`:**
- Use the existing `extractFrontmatter` import to parse YAML frontmatter
- Check that frontmatter exists (content starts with `---\n` and has closing `---`)
- Check that parsed frontmatter contains a `name` field (string, non-empty)
- On success: return `{ valid: true, name: frontmatter.name }`
- On failure: return `{ valid: false, error: 'descriptive message', line: lineNumber }`
  - If no frontmatter delimiters: `{ valid: false, error: 'No YAML frontmatter found (missing --- delimiters)', line: 1 }`
  - If no name field: `{ valid: false, error: 'Missing required "name" field in YAML frontmatter', line: null }`
  - If frontmatter parse error: `{ valid: false, error: 'Malformed YAML frontmatter', line: errorLine }`

**`sanitizeAgentContent(content)`:**
- Build a generic sanitization layer (not just known patterns — per CONTEXT.md decision)
- Replace known editor name variants: replace the literal string 'opencode' (case-insensitive, whole word) with 'OC' ONLY in contexts where it refers to the editor (not in file paths like `.opencode/agents/`). Specifically target patterns like "OpenCode agent", "opencode config" — but NOT path segments. Safest approach: target the pattern where 'opencode' is preceded by a space or start-of-line and followed by a space or end-of-line, excluding path-like contexts (preceded by `/` or `.`).
- Strip structural injection markers: remove lines containing `<system>`, `</system>`, `[INST]`, `[/INST]`, triple-backtick `system` blocks
- Return the sanitized content string
- When mangling detected: auto-fix silently (per CONTEXT.md — no user action needed)

**`generateUnifiedDiff(textA, textB, labelA, labelB)`:**
- Implement a simple LCS-based unified diff algorithm (zero dependencies)
- Split both texts into line arrays
- Compute LCS (longest common subsequence) using standard O(mn) DP — agent files are <500 lines so this is fine
- Generate unified diff output with:
  - `--- labelA` and `+++ labelB` headers
  - `@@ -startA,countA +startB,countB @@` hunk headers
  - Context lines (prefix space), removed lines (prefix `-`), added lines (prefix `+`)
  - Use 3 lines of context around changes (standard unified diff)
- Return the diff as a string. Return empty string if no differences.

Export all three functions from module.exports.

**In src/commands/agent.js — add `cmdAgentListLocal(cwd, raw)`:**

2. Resolve local agents dir: `path.join(cwd, '.opencode', 'agents')`
3. Scan global agents using existing `scanAgents(agentsDir)` to get list of all global agent files
4. Scan local agents dir: if it exists, read `.md` files from it
5. Build a result array with one entry per global agent:
   ```
   {
     name: 'bgsd-executor',
     scope: 'global' | 'local-override',
     drift: null | 'in-sync' | 'drifted'
   }
   ```
   - `scope`: 'local-override' if a file with the same name exists in `.opencode/agents/`, otherwise 'global'
   - `drift`: null for global-only agents. For local-override agents: compare file contents of global vs local. If identical → 'in-sync', if different → 'drifted'. Use simple string comparison (no need for diff here).
6. Output the result using the existing `output()` helper with `raw` flag support
7. For human-readable (non-raw) output, format as a columnar table:
   ```
   Name                  Scope           Drift

   bgsd-planner          local-override  ✓
   bgsd-verifier         local-override  Δ
   ```
   - Drift column: `✓` for in-sync, `Δ` for drifted, empty for global-only
8. Export `cmdAgentListLocal` from module.exports
