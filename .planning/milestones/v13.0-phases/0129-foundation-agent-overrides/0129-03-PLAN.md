  <files>src/commands/agent.js, src/router.js</files>
  <action>
**In src/commands/agent.js — add `cmdAgentSync(cwd, raw, args)`:**

1. Extract `<name>` from args[0]. If missing, error with usage: `'Usage: agent sync <name>'`
2. Strip `.md` extension if provided
3. Resolve global and local agent paths (same pattern as agent:diff)
4. If local override does not exist:
   - Error: `'No local override for "${name}". Create one with: agent override ${name}'`
5. If global agent does not exist:
   - Error: `'Global agent "${name}" not found'`
6. Read both files
7. If contents are identical:
   - Silent exit (per CONTEXT.md: exit code 0, no output)
   - Raw mode: `{ agent: name, action: 'none', identical: true }`
8. Generate diff using `generateUnifiedDiff(globalContent, localContent, 'global', 'local')`
9. Count changed sections — simple heuristic: count `@@` hunk headers in the diff
10. Output sync summary:
    - Human-readable: `'{hunkCount} section(s) modified in upstream. Diff:\n{diff}\n\nAccept upstream changes? (accept/reject)'`
    - Raw mode: `{ agent: name, action: 'pending', sections_modified: hunkCount, diff: diffString }`
11. **Accept handling (for CLI execution):**
    - Since bgsd-tools.cjs is non-interactive (no stdin prompts), the accept/reject is handled via a flag:
    - If args includes `--accept`: apply the sync (overwrite local with sanitized global)
    - If args includes `--reject`: exit silently
    - If neither flag: show the diff summary and instruct user to re-run with `--accept` or `--reject`
12. On accept:
    - Read global content
    - Apply `sanitizeAgentContent(globalContent)` before writing
    - Also apply `validateAgentFrontmatter()` — if global lacks name:, inject it (same logic as override)
    - Write sanitized content to local path
    - Output: `{ agent: name, action: 'accepted', path: localPath }`
    - Human-readable: `'Synced: ${localPath}'`

Export `cmdAgentSync` from module.exports.

**In src/router.js:**

- Update unknown subcommand error message to include 'sync'

**In src/plugin/command-enricher.js — add `local_agent_overrides` enrichment:**

After the existing enrichment fields are populated (around the block that sets `enrichment.milestone`, near line 91), add a new try/catch block:

```javascript
// LOCAL-07: Expose local agent overrides in bgsd-context
try {
  const localAgentsDir = join(resolvedCwd, '.opencode', 'agents');
  if (existsSync(localAgentsDir)) {
    const localAgentFiles = readdirSync(localAgentsDir)
      .filter(f => f.endsWith('.md'));
    enrichment.local_agent_overrides = localAgentFiles.map(f => f.replace('.md', ''));
  } else {
    enrichment.local_agent_overrides = [];
  }
} catch {
  enrichment.local_agent_overrides = [];
}
```

This follows the same pattern as existing enrichment fields:
- Uses the already-imported `existsSync`, `readdirSync` from 'fs' and `join` from 'path'
- Wrapped in try/catch for resilience (enrichment failure should never break commands)
- Returns agent names without `.md` extension
- Empty array when directory doesn't exist or is empty
