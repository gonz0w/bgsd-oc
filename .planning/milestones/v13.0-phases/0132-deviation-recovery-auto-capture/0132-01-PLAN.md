**2. Add `cmdDeviationCapture` function in `src/commands/lessons.js`:**

Add a new function after the existing `cmdLessonsCapture` (around line 153):

```
function cmdDeviationCapture(cwd, options, raw) {
```

**Parameters via `options`:** `rule` (number), `failureCount` (number), `behavioralChange` (string), `agent` (string).

**Logic:**

1. **Rule-1 filter:** Parse `options.rule` as integer. If rule !== 1, output `{ captured: false, reason: 'rule_filtered', rule }` and return immediately. This ensures Rule-2 (missing critical), Rule-3 (blocking/environmental), and Rule-4 (architectural) deviations are never captured.

2. **Validate required fields:** `behavioralChange` and `agent` must be non-empty strings. If missing, output `{ captured: false, reason: 'missing_fields' }` and return.

3. **Cap check:** Call `readLessonsStore(cwd)` to get all entries. Filter entries where `entry.type === 'deviation-recovery'`. If count >= 3, output `{ captured: false, reason: 'cap_reached', count }` and return silently.

4. **Build lesson entry:** Create entry object:
   - `id`: `crypto.randomUUID()`
   - `date`: `new Date().toISOString()`
   - `title`: `"Deviation recovery: ${options.behavioralChange}".substring(0, 200)`
   - `severity`: `"MEDIUM"`
   - `type`: `"deviation-recovery"` (new type ŌĆö add to LESSON_SCHEMA.type_values array)
   - `root_cause`: `"Rule ${options.rule} deviation recovered after ${options.failureCount || 'unknown'} failures"`
   - `prevention_rule`: `options.behavioralChange`
   - `affected_agents`: `options.agent`
   - `deviation_rule`: `parseInt(options.rule, 10)`
   - `failure_count`: `parseInt(options.failureCount, 10) || 0`
   - `behavioral_change`: `options.behavioralChange`

5. **Validate:** Call `validateLesson(entry)`. If invalid, output `{ captured: false, reason: 'validation_failed' }` and return.

6. **Store:** Push normalized entry to entries array, call `writeLessonsStore(cwd, entries)`. Do SQLite dual-write (best-effort, wrapped in try/catch like `cmdLessonsCapture` does).

7. **Output:** `{ captured: true, id, title, type: 'deviation-recovery', entry_count, deviation_rule, failure_count, agent }`.

**Important:** Add `'deviation-recovery'` to `LESSON_SCHEMA.type_values` array (line 15) so validation passes.

**Export:** Add `cmdDeviationCapture` to the module.exports at the bottom of the file.
