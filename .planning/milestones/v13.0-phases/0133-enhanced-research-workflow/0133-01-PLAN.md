<task type="auto">
  <name>Task 1: Implement cmdResearchScore and cmdResearchGaps in research.js</name>
  <files>src/commands/research.js</files>
  <action>
Add the following to `src/commands/research.js` before the `module.exports` line:

1. **`parseResearchFile(content)` helper** — parses a RESEARCH.md string into structured data:
   - Extract header confidence via `/\*\*Confidence:\*\*\s*(HIGH|MEDIUM|LOW)/i`
   - Extract research date via `/\*\*Researched?:\*\*\s*(\d{4}-\d{2}-\d{2})/`
   - Split `## Sources` by `### Primary`, `### Secondary`, `### Tertiary` headings; count bullet items (`- ` lines) per tier
   - Detect official docs: check if any Primary/Secondary source contains URLs from `docs.*` domains or the word "official" in description
   - Extract confidence breakdown from `## Metadata` → `**Confidence breakdown:**` — parse lines matching `/^\s*-\s+(.+?):\s*(HIGH|MEDIUM|LOW)/i`
   - Detect gaps: sections where confidence breakdown shows LOW, plus any `## Open Questions` items → produce `{ gap, severity, section, suggestion }` objects
   - Detect conflicts: scan for language patterns indicating source disagreement (`contradicts`, `disagrees`, `conflicts with`, `unlike .* which says`, `however .* states`) near source citations or within Sources section → produce `{ claim, source_a, source_b }` objects
   - All extractions use defensive fallbacks (null, 0, empty arrays)

2. **`computeConfidenceLevel(parsed)` helper** — composite scoring:
   - Count negative signals: `totalSources < 3`, `!hasOfficialDocs`, research older than 90 days, `conflicts.length > 0`, `highConfidencePct < 30`
   - Return: `>= 3` negatives → 'LOW', `>= 1` → 'MEDIUM', else 'HIGH'
   - Per CONTEXT.md: LOW requires multiple weak factors, not a single red flag

3. **`cmdResearchScore(cwd, args, raw)` handler**:
   - Validate args[0] exists, error if not: `'Usage: research:score <path-to-RESEARCH.md>'`
   - Resolve file path relative to cwd
   - Read file content with `safeReadFile` or `fs.readFileSync` with try/catch
   - Call `parseResearchFile(content)`
   - Call `computeConfidenceLevel(parsed)`
   - Build profile object: `{ source_count, high_confidence_pct (rounded to 1 decimal), oldest_source_days, has_official_docs, confidence_level, flagged_gaps, conflicts }`
   - Write cache: `path.join(path.dirname(resolvedPath), 'research-score.json')` via `fs.writeFileSync`
   - Output: `output(profile, { formatter: formatResearchScore, raw })`
   - No backward-compatible `overall_grade` field (clean break per CONTEXT.md)

4. **`formatResearchScore(profile)` formatter** — for TTY display:
   - Use `banner()`, `sectionHeader()`, `color` from format.js
   - Show confidence level with color (HIGH=green, MEDIUM=yellow, LOW=red)
   - Show source count, high confidence %, oldest source days, has official docs
   - Show gaps table if any, conflicts list if any

5. **`cmdResearchGaps(cwd, args, raw)` handler**:
   - Accept args[0] as either a RESEARCH.md path or research-score.json path
   - Resolve the cache path: if input ends with `-RESEARCH.md` or `.md`, look for `research-score.json` in same directory
   - If cache doesn't exist, error: `'No cached profile found. Run research:score first.'`
   - Read and parse the JSON cache
   - Output: `output({ flagged_gaps: profile.flagged_gaps }, { formatter: formatResearchGaps, raw })`

6. **`formatResearchGaps(result)` formatter** — for TTY display:
   - List each gap with severity, section, and suggestion

7. **Update `module.exports`** — add `cmdResearchScore`, `cmdResearchGaps`, `parseResearchFile`, `computeConfidenceLevel` (export internals for testing)
