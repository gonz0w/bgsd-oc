## Integration Gaps Detail

### GAP-001: storeSessionBundleBatch is Dead Code
**Severity:** Blocker  
**Affected REQs:** STATE-02, BUNDLE-01, BUNDLE-02  
**Phase:** 203  
**Detail:** `storeSessionBundleBatch` at planning-cache.js:1316 is defined with full BEGIN/COMMIT/ROLLBACK transaction support but has **zero callers**. The execute-plan workflow does not invoke any batch state API.

### GAP-002: canBatch Guard is Dead Code
**Severity:** Blocker  
**Affected REQs:** STATE-03  
**Phase:** 203  
**Detail:** `canBatch()` at state-session-mutator.js:20 checks `!SACRED_DATA_STORES.has(store)` but is never called — there is no routing logic that decides between batch vs single-write paths.

### GAP-003: Kahn Sort Never Triggered
**Severity:** Blocker  
**Affected REQs:** PARALLEL-02  
**Phase:** 202  
**Detail:** `resolvePhaseDependencies` is registered in DECISION_REGISTRY at decision-rules.js:944-953, but `enrichment.phases` is never set, so the rule never fires. Wave ordering falls back to frontmatter `wave` field instead of computing from `depends_on`.

### GAP-004: Mutex Primitives are Dead Code
**Severity:** Blocker  
**Affected REQs:** PARALLEL-01  
**Phase:** 202  
**Detail:** `getMutexValue()` and `invalidateMutex()` at planning-cache.js:169-181 and 325-342 are fully implemented with CAS primitives, but have **zero callers**. Parallel stages do not use mutex-protected cache entries.

### GAP-006: Telemetry Gap in Parallel Path
**Severity:** Warning  
**Affected REQs:** ACCEL-02  
**Phase:** 201, 202  
**Detail:** `telemetryLog` hooks are wired in `orchestration.js` but `fanInParallelSpawns` uses `child_process.spawn` directly, bypassing `orchestration.js`. Parallel stage execution does not contribute to telemetry.

## Flow Gaps Detail

### FLOW-001: Parallel Wave Execution Bypasses Kahn Sort
**Phase:** 202  
**Detail:** `fanInParallelSpawns` executes plans in parallel waves, but wave ordering is determined by the frontmatter `wave` field, not by `resolvePhaseDependencies` Kahn sort based on `depends_on`. This means parallel wave ordering is statically defined, not dynamically computed.

### FLOW-002: Batch State API Never Invoked
**Phase:** 203  
**Detail:** The `execute-plan.md` workflow does not call `storeSessionBundleBatch`. Batch transaction support for non-sacred state mutations exists as infrastructure but is not wired into any execution path.
