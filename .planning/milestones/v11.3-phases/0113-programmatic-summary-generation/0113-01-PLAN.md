  <name>Task 1: Implement cmdSummaryGenerate in misc.js with router registration</name>
  <files>src/commands/misc.js, src/router.js, src/lib/commandDiscovery.js</files>
  <action>
Add `cmdSummaryGenerate(cwd, phaseArg, planArg, raw)` to `src/commands/misc.js`. The function follows this pipeline:

1. **Resolve inputs:** Use `findPhaseInternal(cwd, phaseArg)` to get phase directory. Construct PLAN.md path from `{padded_phase}-{padded_plan}-PLAN.md`. Validate both exist.

2. **Extract PLAN.md data:** Use `extractFrontmatter()` on PLAN.md to get phase, plan, requirements, files_modified, type. Extract task names from body using regex: `/<name>Task \d+:\s*(.+?)<\/name>/g`.

3. **Extract git commits:** Use `structuredLog(cwd, { count: 30 })` and filter by conventional commit scope matching `{phase_number}-{padded_plan}` (accept both zero-padded and unpadded forms, e.g., both `0113-01` and `113-01`). If no scoped commits found, fall back to last N commits as-is.

4. **Extract file diffs:** If scoped commits found, use `diffSummary(cwd, { from: lastCommit.hash + '~1', to: firstCommit.hash })` (note: structuredLog returns newest-first, so lastCommit is the earliest). If no commits, skip diff section.

5. **Extract timing:** Use git commit timestamps — first scoped commit date as start, last scoped commit date as end. Calculate duration in minutes. Fall back to "TODO: duration" if no commits.

6. **Infer frontmatter fields:**
   - `subsystem`: Most common top-level directory among changed files (e.g., `src/commands/` → `cli`, `src/plugin/` → `plugin`, `src/lib/` → `core`, `tests/` → `testing`, `workflows/` → `workflow`). Default to first directory name if no mapping matches.
   - `tags`: Unique file extensions mapped to tech names (`.js` → `javascript`, `.cjs` → `commonjs`, `.md` → `markdown`, `.json` → `json`). Plus any library names from PLAN.md frontmatter tags if present.
   - `key-files.created`: Files in diff with 0 deletions (new files). `key-files.modified`: Top 3 files by total changes (insertions + deletions).
   - `requirements-completed`: Copy directly from PLAN.md `requirements` frontmatter field.

7. **Generate markdown:** Build the full SUMMARY.md content using template literals (NOT reconstructFrontmatter for the body — only for the YAML frontmatter block). Use `reconstructFrontmatter()` for the `---` delimited frontmatter section. Body follows templates/summary.md format exactly:
   - `# Phase {N} Plan {NN}: {plan_objective} Summary` heading
   - `**TODO: one-liner (substantive one-liner — NOT "phase complete")**` bold line
   - `## Performance` with duration, start, end, task count, file count
   - `## Accomplishments` with `TODO: accomplishments (2-3 bullets, what changed and why)`
   - `## Task Commits` listing each task name + commit hash + type
   - `## Files Created/Modified` listing each file with `[+ins/-del]`
   - `## Decisions Made` with `TODO: decisions (key decisions with brief rationale, or "None - followed plan as specified")`
   - `## Deviations from Plan` with `TODO: deviations (describe any deviations, or "None - plan executed exactly as written")`
   - `## Lessons Learned` with `TODO: lessons (what would you do differently, or "None")`
   - `## Issues Encountered` with `TODO: issues (problems and how resolved, or "None")`
   - `## Next Phase Readiness` with `TODO: next-phase-readiness (what's ready for next phase)`

8. **Merge/preserve on re-run:** If SUMMARY.md already exists at the target path:
   - Read existing content
   - For each judgment section (Accomplishments, Decisions Made, Deviations from Plan, Lessons Learned, Issues Encountered, Next Phase Readiness, and the one-liner bold line), check if the section text contains `TODO:` (the specific marker pattern)
   - If `TODO:` marker is absent → section was filled by LLM → preserve the filled content from the existing file
   - If `TODO:` marker is present → section is unfilled → regenerate with fresh data
   - Always regenerate data sections (frontmatter, Performance, Task Commits, Files Created/Modified) with latest git data
   - For frontmatter key-decisions and one-liner: check if they contain `TODO:` — if filled, preserve

9. **Validate:** Call the same validation logic used by `cmdVerifySummary` — check that referenced files exist and commit hashes are valid. If validation fails, still write the file but include a warning in the status output.

10. **Write and report:** Write the file to `{phaseDir}/{padded_phase}-{padded_plan}-SUMMARY.md`. Output JSON with `{ path, commits_found, files_found, todos_remaining, warnings }`. The `todos_remaining` count tells the LLM how many sections it needs to fill.

**Router registration:** Add `summary-generate` to `src/router.js` in the util namespace, following the exact pattern of `summary-extract` (router.js ~line 967):

**Exports:** Add `cmdSummaryGenerate` to the `module.exports` object at the bottom of misc.js.

Error handling: Wrap entire function in try/catch. On failure, output `{ error, fallback: true }` — the workflow will fall back to full LLM authorship per CONTEXT.md constraint.
