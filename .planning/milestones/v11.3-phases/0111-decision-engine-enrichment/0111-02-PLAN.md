<task type="auto">
  <name>Task 1: Create decisions CLI command module</name>
  <files>src/commands/decisions.js</files>
  <action>
Create `src/commands/decisions.js` as a CJS module following the `src/commands/audit.js` pattern.

Import `DECISION_REGISTRY` and individual rule functions from `../lib/decision-rules`.
Import `output` from `../lib/output` for structured JSON/TTY dual-mode output.
Import `format` from `../lib/format` for TTY formatting (tables, banners, colors).

**Three command handlers:**

1. `cmdDecisionsList(cwd, args, raw)` — `decisions:list`
   - Lists all registered decision rules in a formatted table
   - Columns: rule_id, category, confidence_range, description
   - Groups by category with section headers
   - Summary line: total rules, categories, candidate coverage count
   - JSON mode: outputs DECISION_REGISTRY array directly
   - Create `formatDecisionsList(data)` TTY formatter following the `formatAuditScan` pattern (banner → table → summary)

2. `cmdDecisionsInspect(cwd, args, raw)` — `decisions:inspect <rule_id>`
   - Takes rule_id as first positional arg
   - Shows full rule details: id, name, category, description, inputs, outputs, confidence_range
   - If rule not found, error with available rule IDs
   - JSON mode: outputs the registry entry directly
   - Create `formatDecisionsInspect(data)` TTY formatter

3. `cmdDecisionsEvaluate(cwd, args, raw)` — `decisions:evaluate <rule_id> [--state '{json}']`
   - Takes rule_id as first positional arg
   - Parses `--state` flag for JSON state input
   - If no --state provided, attempt to build state from current project context (using getProjectState pattern from enricher)
   - Runs the rule's resolve function against the state
   - Returns the decision result `{value, confidence, rule_id, metadata?}`
   - Error if rule_id not found or state parse fails
   - JSON mode: outputs result directly
   - Create `formatDecisionsEvaluate(data)` TTY formatter

Each formatter follows the enricher convention: use `output(result, { formatter: formatFn })` from output.js.

  </verify>

- Each handler supports JSON and TTY output modes
- cmdDecisionsInspect validates rule_id and shows helpful error with available IDs
- cmdDecisionsEvaluate parses --state JSON and runs rule resolution
