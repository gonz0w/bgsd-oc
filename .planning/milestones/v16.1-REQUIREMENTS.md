## Category: TDD (TDD Reliability)

### Requirements

- [x] **TDD-01:** Deterministic TDD selection - Planner evaluates TDD eligibility for every implementation plan even when the roadmap omits a `**TDD:**` hint

- [x] **TDD-02:** TDD decision rationale - Each implementation plan records a short rationale explaining why TDD was selected or intentionally skipped

- [x] **TDD-03:** Severity-aware TDD checking - Checker enforces `required`, `recommended`, and omitted TDD hints with blocker, warning, and informational behavior respectively

- [x] **TDD-04:** Canonical TDD contract - One source of truth defines TDD structure, terminology, command names, and expected artifacts across docs, workflows, skills, and CLI output

- [x] **TDD-05:** Semantic RED/GREEN/REFACTOR validation - `execute:tdd` verifies intended TDD state transitions using TDD-aware semantics instead of raw exit codes alone

- [x] **TDD-06:** End-to-end TDD proof - Fixture-backed integration coverage proves a real `type: tdd` plan executes end-to-end and leaves the expected commit and summary audit trail

## Category: FLOW (Workflow Acceleration)

### Requirements

- [x] **FLOW-01:** Shared phase snapshot - A compact `phase:snapshot <phase>` command returns phase metadata, requirements, artifact paths, plan index, and related execution context without repeated discovery calls

- [x] **FLOW-02:** Batched plan finalization - A single command atomically advances plan position, updates progress, records metrics and decisions, and writes continuity data for plan completion

- [x] **FLOW-03:** Reused init discovery - High-traffic init flows reuse cached phase-tree and artifact discovery data instead of rescanning directories repeatedly

- [x] **FLOW-04:** Fast discuss mode - `discuss-phase --fast` batches low-risk clarification turns while preserving locked decisions, agent discretion, and deferred ideas

- [x] **FLOW-05:** Batch verify mode - `verify-work --batch N` presents routine verification checks in configurable batches while keeping one-at-a-time verification as the default path

- [x] **FLOW-06:** Cached plan indexing - Plan indexing uses PlanningCache-backed reads instead of reparsing markdown plans on every invocation

- [x] **FLOW-07:** Durable handoff artifacts - A deterministic phase handoff JSON file and update command preserve compact cross-step state for fresh-context chaining

- [x] **FLOW-08:** Fresh-context delivery pipeline - A chained phase-delivery command executes discuss, research, plan, execute, and verify through resumable handoffs without relying on one long-lived chat transcript
