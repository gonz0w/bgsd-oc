Update src/plugin/parsers/state.js to read from SQLite first, falling back to markdown parsing.

**1. Add SQLite-first path to parseState():**

```javascript
import { getDb, PlanningCache } from '../lib/db-cache.js';
```

Before the existing readFileSync(statePath) call, attempt SQLite read:
- Get cache: `const db = getDb(resolvedCwd); const cache = new PlanningCache(db);`
- If `db.backend === 'sqlite'`:
  - Call `cache.getSessionState(resolvedCwd)` to get position data
  - Call `cache.getSessionContinuity(resolvedCwd)` for continuity
  - If session_state row exists (non-null return):
    - Build the frozen result object from SQL data instead of parsing markdown
    - The `raw` field should still contain the STATE.md content (read it for consumers that expect raw markdown)
    - The `phase`, `currentPlan`, `status`, `lastActivity`, `progress` fields come from SQLite columns
    - The `getField(name)` and `getSection(name)` methods should query SQLite for known fields (phase_number, current_plan, status, last_activity) and fall back to parsing raw for unknown fields
    - Cache and return the frozen result
- If SQLite returns null (no migration yet), fall through to existing markdown parsing

**2. Add accumulated context query support:**

Add new methods to the frozen state object for SQLite-backed queries:
- `getDecisions(options)` → calls `cache.getSessionDecisions(cwd, options)` — returns array or null
- `getTodos(options)` → calls `cache.getSessionTodos(cwd, options)` — returns array or null
- `getBlockers(options)` → calls `cache.getSessionBlockers(cwd, options)` — returns array or null
- `getMetrics(options)` → calls `cache.getSessionMetrics(cwd, options)` — returns array or null

These methods enable SES-03: "view accumulated context from SQLite without parsing STATE.md."

If on Map backend, these methods return null and callers fall back to parsing STATE.md sections.
