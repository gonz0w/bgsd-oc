  <name>Task 1: Extend PlanningCache with enrichment query methods and refactor enricher to zero duplication</name>
  <files>src/plugin/lib/db-cache.js, src/plugin/command-enricher.js</files>
  <action>
**Part A — PlanningCache enrichment queries (`src/plugin/lib/db-cache.js`):**

Add three new methods to the `PlanningCache` class:

1. `getSummaryCount(phaseNumber, cwd)` — Query the `plans` table: count plans for the phase, then for each plan path, check if a corresponding SUMMARY file exists in `file_cache` or via fs.existsSync. Returns `{ planCount, summaryCount, summaryFiles }`. On Map backend, return `null`.

2. `getIncompletePlans(phaseNumber, cwd)` — Query `plans` table for phase, check each plan path for matching SUMMARY file. Returns array of incomplete plan filenames (no matching SUMMARY). On Map backend, return `null`.

3. `getLastSummaryContent(phaseDir)` — Read the last summary file (sorted) in the phase dir. Returns `{ hasUnresolvedIssues, hasBlockers }` or `null`. This is a thin helper, not SQLite-backed — it's called at most once.

**Part B — Eliminate duplication in `command-enricher.js`:**

Refactor `enrichCommand` to compute each derived value exactly ONCE:

1. Move `parsePlans` call to a single location early in the function. Store result in a local `plans` variable. All subsequent code references this variable — no second `parsePlans` call.

2. Move `listSummaryFiles` call to a single location. Store result in a local `summaryFiles` variable. All 3 current call sites reference this cached result.

3. Add SQLite-first path: Before calling `parsePlans` or `listSummaryFiles`, try `PlanningCache.getSummaryCount()` and `PlanningCache.getIncompletePlans()`. If SQLite returns non-null, use those results directly — skip `parsePlans` and `listSummaryFiles` entirely for those data points.

4. Keep the existing `parsePlans` call as the fallback when SQLite returns null (cold cache / Map backend).

5. For the task_types extraction (line ~196), reuse the already-loaded `plans` variable instead of calling `parsePlans` a third time.

**Critical constraint:** The enrichment output JSON shape must be IDENTICAL before and after refactoring. Same field names, same value types, same behavior for missing data.
