<objective>
Wire existing plugin-layer parsers (roadmap.js, plan.js) to use PlanningCache as a SQLite-first data source with transparent write-through caching and mtime-based invalidation.

Purpose: Make parsed planning data persist across CLI invocations — queries replace markdown re-parsing when cache is valid. This is the integration that fulfills TBL-01 through TBL-04 for end users.
Output: Modified parsers that check SQLite first, fall back to markdown parsing on miss/stale, and write through on every parse.
</objective>

  <action>
Modify `parseRoadmap(cwd)` to implement the SQLite-first pattern:

1. **Import PlanningCache and getDb** at the top of the file. Since this is an ESM file (`export function`), use a dynamic `require()` or a lazy import pattern to avoid circular dependencies with the CJS db.js module. The cleanest approach: use a lazy getter that loads on first call.

2. **Before the existing Map cache check**, add SQLite freshness check:
   ```
   const roadmapPath = join(resolvedCwd, '.planning', 'ROADMAP.md');
   const cache = getPlanningCache(resolvedCwd);
   if (cache) {
     const freshness = cache.checkFreshness(roadmapPath);
     if (freshness === 'fresh') {
       // Reconstruct the frozen roadmap object from SQLite data
       const phases = cache.getPhases(resolvedCwd);
       const milestones = cache.getMilestones(resolvedCwd);
       const progress = cache.getProgress(resolvedCwd);
       if (phases && phases.length > 0) {
         // Build result object with same shape as markdown-parsed version
         // Include getPhase(), getMilestone(), currentMilestone accessors
         const result = buildRoadmapFromCache(phases, milestones, progress);
         _cache.set(resolvedCwd, result);
         return result;
       }
     }
   }
   ```

3. **After parsing markdown** (existing code), add write-through:
   ```
   // Write through to SQLite cache
   if (cache) {
     cache.storeRoadmap(resolvedCwd, roadmapPath, result);
   }
   ```

4. **Create `buildRoadmapFromCache(phases, milestones, progress)`** helper that reconstructs the same frozen object shape from SQLite rows. Must include:
   - `raw: null` (raw markdown not stored for cache hits — not needed for most consumers)
   - `milestones` array of frozen objects
   - `phases` array of frozen objects
   - `progress` array of frozen objects
   - `getPhase(num)` method that queries SQLite or uses the cached phases array
   - `getMilestone(name)` method
   - `currentMilestone` getter

5. **Handle requirements extraction** during storeRoadmap: parse the `**Requirements**: REQ-01, REQ-02` line from each phase section and store as individual requirement rows.

6. **Preserve existing invalidateRoadmap()** — also clear SQLite cache entries for the roadmap file.

IMPORTANT: The reconstructed object from cache MUST have the same shape as the markdown-parsed version. Consumers should not be able to tell whether data came from SQLite or markdown. The only acceptable difference is `raw` being null on cache hits (consumers that need raw markdown will still work because the in-memory Map cache stores the full object with raw after the first markdown parse in a session).
