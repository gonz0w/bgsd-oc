<task type="auto">
  <name>Task 2: Implement 5 new decision functions and expand plan-existence-route</name>
  <files>src/lib/decision-rules.js</files>
  <action>
Add the following functions and registry entries to `src/lib/decision-rules.js`:

1. **resolveModelSelection(state)** — Category: `configuration`
   - Inputs: `agent_type`, `model_profile` (tier name), `db` (database handle, may be null)
   - Logic: If `db` is available, try PlanningCache.getModelProfile(). If row exists, use override_model if set, else use the tier column matching model_profile. Fall back to static MODEL_PROFILES from constants.js if no DB or no row.
   - Returns: `{ value: { tier, model }, confidence: 'HIGH', rule_id: 'model-selection' }`
   - Import PlanningCache at top of file (lazy require to avoid circular deps)

2. **resolveVerificationRouting(state)** — Category: `workflow-routing`
   - Inputs: `task_count`, `files_modified_count`, `has_test_command`, `verifier_enabled`
   - Logic: If verifier disabled → 'skip'. If task_count <= 2 AND files_modified_count <= 4 → 'light'. Else → 'full'.
   - Returns: `{ value: 'full'|'light'|'skip', confidence: 'HIGH', rule_id: 'verification-routing' }`

3. **resolveResearchGate(state)** — Category: `workflow-routing`
   - Inputs: `research_enabled`, `has_research`, `has_context`, `phase_has_external_deps`
   - Logic: If research disabled → { run: false, depth: null }. If has_research already → { run: false, depth: null }. If phase_has_external_deps → { run: true, depth: 'deep' }. Else if no context → { run: true, depth: 'quick' }. Else → { run: false, depth: null }.
   - Returns: `{ value: { run, depth }, confidence: 'HIGH', rule_id: 'research-gate' }`

4. **Expand resolvePlanExistenceRoute(state)** — existing function
   - Add new inputs: `deps_complete`, `has_blockers` (alongside existing `plan_count`, `has_research`, `has_context`)
   - Add new return values before the existing logic:
     - If `plan_count > 0 && has_blockers` → 'blocked-deps'
     - If `plan_count > 0 && deps_complete === false` → 'blocked-deps'
     - If `plan_count > 0 && has_context` → 'ready' (upgraded from plain 'has-plans')
     - Keep 'has-plans' for plan_count > 0 without context (backward compat)
     - If `!has_context && !has_research` → 'missing-context' (before 'needs-research')
   - Update registry entry inputs array to include `deps_complete`, `has_blockers`

5. **resolveMilestoneCompletion(state)** — Category: `state-assessment`
   - Inputs: `phases_total`, `phases_complete`, `has_incomplete_plans`, `milestone_name`
   - Logic: If phases_complete === phases_total AND !has_incomplete_plans → { ready: true, action: 'complete' }. If phases_complete === phases_total - 1 → { ready: false, action: 'finish-last-phase' }. Else → { ready: false, action: 'continue' }.
   - Returns: `{ value: { ready, action }, confidence: 'HIGH', rule_id: 'milestone-completion' }`

6. **resolveCommitStrategy(state)** — Category: `execution-mode`
   - Inputs: `task_count`, `plan_type`, `files_modified_count`, `is_tdd`
   - Logic: Granularity: if is_tdd → 'per-phase' (TDD commits at red/green/refactor). If task_count <= 1 → 'per-plan'. Else → 'per-task'. Prefix: if plan_type === 'tdd' → 'test'. If files_modified_count === 0 → 'docs'. Else infer from file patterns (test files → 'test', docs → 'docs', else 'feat').
   - Returns: `{ value: { granularity, prefix }, confidence: 'HIGH'|'MEDIUM', rule_id: 'commit-strategy' }`

Add all 5 new functions to module.exports. Add all 6 entries (5 new + 1 updated) to DECISION_REGISTRY. Ensure the expanded plan-existence-route preserves all old behavior for old callers (no new inputs → old behavior).
