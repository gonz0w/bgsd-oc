# Plan 02: Workflow Structure Verification

## Objective

Add `workflow:verify-structure` command that validates all workflows preserve their structural elements (Task() calls, CLI commands, section markers, question blocks) — comparing against a baseline snapshot to detect regressions with explicit pass/fail output.

must_haves:

## Tasks

### Task 1: Implement workflow:verify-structure command

- **Type:** auto
- **Files:**
  - `src/commands/workflow.js` (modify)

- **Action:**

  Add `cmdWorkflowVerifyStructure(cwd, args, raw)` to `src/commands/workflow.js` and export it. The command compares a baseline snapshot's structural fingerprints against the current workflows to detect regressions.

  **Input:** `workflow:verify-structure [baseline-path]`
  - If `baseline-path` given: use it as the reference snapshot
  - If omitted: find the most recent `workflow-baseline-*.json` in `.planning/baselines/`
  - If no baseline exists: error with "No baseline found. Run `workflow:baseline` first."

  **Comparison logic:**
  1. Load the baseline snapshot (which includes `structure` per workflow from Plan 01's `cmdWorkflowBaseline`)
  2. For each workflow in the baseline, read the current file and extract its current structural fingerprint (reuse the `extractStructuralFingerprint()` helper created in Plan 01)
  3. Compare baseline vs current per element category:
     - **Task() calls:** Each `Task(` invocation string from baseline must still exist in current. Report any missing.
     - **CLI commands:** Each `bgsd-tools <namespace:command>` from baseline must still exist. Report any missing.
     - **Section markers:** Each `<!-- section: name -->` from baseline must still exist. Report any missing.
     - **Question blocks:** Each `<question>` from baseline must still exist. Report any missing.
     - **XML tags:** Each structural tag (`<step>`, `<process>`, etc.) from baseline must still exist. Report any missing.
  4. Track pass/fail per workflow: workflow passes if ALL structural elements are preserved (additions are fine, removals are failures)
  5. A workflow removed entirely = failure

  **Output format:**
  ```json
  {
    "baseline_file": "path/to/baseline",
    "baseline_date": "2026-03-16T...",
    "verified_at": "2026-03-16T...",
    "summary": {
      "total_workflows": 44,
      "passed": 42,
      "failed": 2,
      "skipped": 0
    },
    "results": [
      {
        "name": "execute-plan.md",
        "status": "pass",
        "missing": []
      },
      {
        "name": "plan-phase.md",
        "status": "fail",
        "missing": [
          { "type": "cli_command", "value": "bgsd-tools plan:phase" },
          { "type": "task_call", "value": "Task(description=\"Research\"...)" }
        ]
      }
    ]
  }
  ```

  **Stderr output:** Human-readable summary table with pass/fail per workflow. Failed workflows show their missing elements indented below. Final line: `PASS: X/Y workflows` or `FAIL: X regressions detected in Y workflows`.

  **Exit code:** Use `process.exitCode = 1` if any workflow fails (not `process.exit(1)` which would skip cleanup). This allows test harnesses and CI to detect failures.

- **Verify:** After building (`npm run build`), run `workflow:baseline --raw` to create a reference, then `workflow:verify-structure --raw` — should pass with all workflows clean. Manually test regression detection by temporarily modifying a workflow file, running verify-structure, and confirming it reports the missing element.

- **Done:** `workflow:verify-structure` compares current workflows against a baseline, reporting missing structural elements as explicit failures with per-workflow pass/fail and summary stats.

### Task 2: Add tests for workflow:verify-structure

- **Type:** auto
- **Files:**
  - `tests/workflow.test.cjs` (modify)

- **Action:**

  Add a new `describe('workflow:verify-structure')` block to `tests/workflow.test.cjs`.

  **Unit tests:**
  - Test comparison logic with mock baseline and current fingerprints:
    - All elements preserved → pass
    - One Task() call removed → fail with correct `missing` entry
    - One CLI command removed → fail
    - Section marker removed → fail
    - Question block removed → fail
    - New elements added (not in baseline) → still pass (additions are fine)
    - Workflow removed entirely → fail
    - Workflow added (not in baseline) → pass (skip, not a regression)
  - Test edge cases:
    - Empty baseline (no structural elements) → all pass
    - Workflow with no structural elements at all → pass

  **Integration tests:**
  - `workflow:verify-structure --raw` with a valid baseline → JSON output with `summary.passed` matching workflow count
  - `workflow:verify-structure nonexistent.json --raw` → error message
  - Verify exit code behavior: create a temp baseline, modify a temp workflow, verify exit code is non-zero

  Target: 10-12 test cases.

- **Verify:** `npm test` passes with all new tests green. `node --test tests/workflow.test.cjs` runs independently.

- **Done:** Comprehensive test coverage for verify-structure including pass cases, regression detection for each element type, edge cases, and integration tests.
