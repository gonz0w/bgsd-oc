# Plan 01: Workflow Baseline & Compare Commands

## Objective

Create the `workflow:` CLI namespace with `baseline` and `compare` subcommands so users can measure per-workflow token counts and diff two snapshots to see token deltas with total reduction percentage.

## Must-Haves

```yaml
must_haves:
  truths:
    - "User runs `workflow:baseline` and receives a JSON snapshot with per-workflow token counts for all 44 workflows"
    - "Snapshot is saved to .planning/baselines/workflow-baseline-{timestamp}.json"
    - "User runs `workflow:compare <snapshot-a> <snapshot-b>` and sees per-workflow token deltas with total reduction percentage"
    - "Both commands produce human-readable stderr tables and machine-readable JSON stdout"
  artifacts:
    - path: "src/commands/workflow.js"
      provides: "Workflow measurement commands"
      exports: ["cmdWorkflowBaseline", "cmdWorkflowCompare"]
    - path: "src/router.js"
      provides: "workflow namespace routing"
      contains: "case 'workflow'"
    - path: "src/lib/constants.js"
      provides: "COMMAND_HELP entries for workflow:baseline and workflow:compare"
    - path: "tests/workflow.test.cjs"
      provides: "Unit and integration tests for baseline and compare"
  key_links:
    - from: "src/router.js"
      to: "src/commands/workflow.js"

      pattern: "measureAllWorkflows"
```

## Tasks

### Task 1: Create workflow namespace with baseline and compare commands

- **Type:** auto
- **Files:**
  - `src/commands/workflow.js` (create)
  - `src/router.js` (modify)
  - `src/lib/constants.js` (modify)
  - `src/lib/commandDiscovery.js` (modify)

- **Action:**

  1. **Export `measureAllWorkflows` from features.js** — Add it to the existing `module.exports` in `src/commands/features.js` so workflow.js can reuse it without duplicating 90 lines of measurement code.

  2. **Create `src/commands/workflow.js`** with two exported functions:

     **`cmdWorkflowBaseline(cwd, raw)`:**
     - Call `measureAllWorkflows(cwd)` from features.js to get token measurements
     - Additionally, extract a **structural fingerprint** per workflow: count of Task() calls, CLI commands (`bgsd-tools` invocations in code blocks), section markers (`<!-- section: ... -->`), question blocks (`<question>`), and key XML tags (`<step>`, `<process>`, `<purpose>`)
     - Build a snapshot object: `{ version: 1, timestamp, workflow_count, total_tokens, workflows: [{ name, workflow_tokens, ref_count, ref_tokens, total_tokens, structure: { task_calls: [...], cli_commands: [...], section_markers: [...], question_blocks: [...], xml_tags: [...] } }] }`
     - Save to `.planning/baselines/workflow-baseline-{timestamp}.json`
     - Print human-readable table to stderr (workflow name, tokens, structural element counts)
     - Output JSON to stdout via `output(snapshot, raw)`

     **`cmdWorkflowCompare(cwd, args, raw)`:**
     - Accept two positional args: `<snapshot-a>` and `<snapshot-b>` (file paths). If only one arg given, treat it as snapshot-a and measure current state as snapshot-b. If zero args, find two most recent baselines.
     - Load both snapshots from JSON files
     - Build comparison: for each workflow, compute `delta = b.total_tokens - a.total_tokens` and `percent_change = (delta / a.total_tokens) * 100`
     - Handle added/removed workflows
     - Compute summary: `{ before_total, after_total, delta, percent_change, workflows_improved, workflows_unchanged, workflows_worsened }`
     - Print comparison table to stderr with per-workflow deltas
     - Output JSON via `output(result, raw)`

  3. **Wire into router** (`src/router.js`):
     - Add `function lazyWorkflow() { return _modules.workflow || (_modules.workflow = require('./commands/workflow')); }` after line 113
     - Add `'workflow'` to `KNOWN_NAMESPACES` array on line 256
     - Add `case 'workflow': { ... }` switch block before the `default:` case (before line 1459), routing `baseline`, `compare`, and `verify-structure` subcommands
     - Update the default error message to include `workflow` in the namespace list

  4. **Add COMMAND_HELP entries** in `src/lib/constants.js` for `workflow:baseline` and `workflow:compare`

  5. **Add command discovery entries** in `src/lib/commandDiscovery.js`:
     - Add aliases: `'w': 'workflow'`, `'w:b': 'workflow:baseline'`, `'w:c': 'workflow:compare'`, `'w:v': 'workflow:verify-structure'`
     - Add to COMMAND_CATEGORIES under a new `measurement` category
     - Add to COMMAND_TREE

- **Verify:** `node bin/bgsd-tools.cjs workflow:baseline --raw` succeeds and outputs JSON with `workflow_count`, `total_tokens`, and `workflows` array. Saved file exists in `.planning/baselines/`. Build succeeds with `npm run build`.

- **Done:** `workflow:baseline` produces a JSON snapshot with token counts + structural fingerprints for all workflows, saved to disk. `workflow:compare` diffs two snapshots showing per-workflow deltas and total reduction percentage.

### Task 2: Add tests for workflow:baseline and workflow:compare

- **Type:** auto
- **Files:**
  - `tests/workflow.test.cjs` (create)

- **Action:**

  Create `tests/workflow.test.cjs` using Node.js built-in test runner (`node:test` + `node:assert`). Follow the patterns in existing test files (e.g., `tests/verify.test.cjs`).

  **Unit tests (require src modules directly):**
  - Test structural fingerprint extraction: provide sample workflow content with known Task() calls, CLI commands, section markers, question blocks — verify the extracted counts and values
  - Test compare logic: provide two mock snapshots — verify deltas, percent changes, and summary stats are computed correctly
  - Test edge cases: empty workflows, missing workflows in one snapshot (added/removed), zero-token workflows

  **Integration tests (use `runGsdTools` helper):**
  - `workflow:baseline --raw` — verify output parses as JSON, has expected fields, workflow_count >= 40
  - `workflow:compare <a> <b> --raw` with two saved baseline files — verify output has `summary.delta` and `workflows` array
  - `workflow:compare --raw` with no args — verify it finds the most recent baselines or produces meaningful error
  - Error cases: nonexistent file path, invalid JSON file

  Target: 10-15 test cases covering happy paths, edge cases, and error handling.

- **Verify:** `npm test` passes with all new tests green. `node --test tests/workflow.test.cjs` runs independently.

- **Done:** Comprehensive test coverage for baseline and compare commands, including unit tests for internal logic and integration tests for CLI behavior.
