# Plan 01: Baseline Snapshot & Shared Block Extraction

## Objective

Take a pre-compression baseline snapshot using `workflow:baseline` and create new skill files for shared blocks that appear across multiple workflows, so that subsequent compression plans can replace inline content with `<skill:X />` references.

## Must-Haves

```yaml
must_haves:
  truths:
    - "A workflow-baseline-*.json snapshot exists in .planning/baselines/ capturing the pre-compression state of all 44 workflows"
    - "Skill files exist for CI quality gate, research pipeline, and bgsd-context initialization patterns"
    - "Each new skill follows the established SKILL.md format (YAML frontmatter, Purpose, Content with section markers)"
    - "skill-index/SKILL.md is regenerated to include the new skills"
  artifacts:
    - path: "skills/ci-quality-gate/SKILL.md"
      provides: "CI quality gate block shared by execute-phase and quick"
      contains: "CI_ENABLED check, Task() spawn to bgsd-github-ci, ci_parameters"
    - path: "skills/research-pipeline/SKILL.md"
      provides: "Research + synthesize pipeline shared by new-milestone and new-project"
      contains: "4 parallel bgsd-project-researcher spawns, synthesizer spawn, research scoring"
    - path: "skills/bgsd-context-init/SKILL.md"
      provides: "Standard bgsd-context initialization preamble shared by all workflows"
      contains: "bgsd-context detection, plugin-required error message, parse instruction"
  key_links:
    - from: "skills/*/SKILL.md"
      to: "workflows/*.md"
      via: "<skill:X /> reference convention"
      pattern: "<skill:"
```

## Tasks

### Task 1: Take pre-compression baseline snapshot

- **Type:** auto
- **Files:**
  - `.planning/baselines/workflow-baseline-*.json` (created by CLI)

- **Action:**

  1. **Run `workflow:baseline`** to capture token counts and structural fingerprints for all 44 workflows in their current uncompressed state. This becomes the "before" snapshot for Phase 135 measurement.

  2. **Record the snapshot filename** — note the exact path for use in final verification (Plan 05). The snapshot will be auto-saved to `.planning/baselines/workflow-baseline-{timestamp}.json`.

  3. **Verify snapshot integrity** — run `workflow:compare` with the new snapshot vs itself to confirm zero delta (sanity check). Run `workflow:verify-structure` against the snapshot to confirm all 44 workflows pass structural validation.

  4. **Record token counts for the top 10 workflows** to stderr for reference:
     - discuss-phase, execute-phase, new-milestone, execute-plan, transition
     - new-project, audit-milestone, quick, resume-project, map-codebase

- **Verify:** `workflow:baseline --raw` produces JSON with `workflow_count: 44`. `workflow:verify-structure` exits with code 0. Snapshot file exists in `.planning/baselines/`.

- **Done:** Pre-compression baseline snapshot saved. Token counts and structural fingerprints captured for all 44 workflows. Structural verification passes clean.

### Task 2: Create skill files for shared workflow blocks

- **Type:** auto
- **Files:**
  - `skills/ci-quality-gate/SKILL.md` (create)
  - `skills/research-pipeline/SKILL.md` (create)
  - `skills/bgsd-context-init/SKILL.md` (create)
  - `skills/skill-index/SKILL.md` (modify — regenerate)

- **Action:**

  1. **Create `skills/ci-quality-gate/SKILL.md`** — Extract the CI quality gate block that appears nearly identically in `execute-phase.md` and `quick.md`. The skill should contain:
     - YAML frontmatter: `name: ci-quality-gate`, `type: shared`, `agents: [orchestrator]`, `sections: [ci-gate]`
     - Purpose: CI quality gate spawn pattern for post-execution code scanning
     - Content section `<!-- section: ci-gate -->`: The config check (`util:config-get workflow.ci_gate`), Task() spawn to `bgsd-github-ci` with `<ci_parameters>` block, branch naming convention, and the conditional skip logic
     - Parameterize: Use `{{scope}}` for the branch name component and `{{base_branch}}` for the target branch

  2. **Create `skills/research-pipeline/SKILL.md`** — Extract the research + synthesize pipeline from `new-milestone.md` and `new-project.md`. The skill should contain:
     - YAML frontmatter: `name: research-pipeline`, `type: shared`, `agents: [orchestrator]`, `sections: [spawn-researchers, synthesize, score]`
     - Content: The 4 parallel `bgsd-project-researcher` spawns (Stack, Features, Architecture, Pitfalls), the synthesizer Task() spawn, and the `research:score` quality check
     - Parameterize: Use `{{research_context}}` for the milestone/project-specific context block
     - Include both the spawn pattern and the quality scoring/retry loop

  3. **Create `skills/bgsd-context-init/SKILL.md`** — Extract the standard bgsd-context initialization preamble that appears in all 10 workflows. The skill should contain:
     - YAML frontmatter: `name: bgsd-context-init`, `type: shared`, `agents: [all]`, `sections: [init]`
     - Content: The 2-paragraph preamble about `<bgsd-context>` auto-injection, plugin-required error message, and the "Parse <bgsd-context> JSON" instruction pattern
     - Note: Individual workflows still list their specific field requirements after the `<skill:bgsd-context-init />` tag

  4. **Regenerate `skills/skill-index/SKILL.md`** — Run the skill index generator or manually add the 3 new skills to the auto-generated index so they appear in skill discovery.

  5. **Verify format consistency** — Each new skill should match the structure of existing skills (e.g., `deviation-rules/SKILL.md`): YAML frontmatter, `## Purpose`, `## Placeholders`, `## Content` with `<!-- section: -->` markers, `## Cross-references`.

- **Verify:** All 3 new skill directories and SKILL.md files exist. Each has valid YAML frontmatter. `skill-index/SKILL.md` lists all 30 skills (27 existing + 3 new). New skills follow the established format.

- **Done:** Three new skill files created for CI quality gate, research pipeline, and bgsd-context initialization. Skill index updated. All new skills follow established format conventions and are ready for `<skill:X />` references in compression plans.
