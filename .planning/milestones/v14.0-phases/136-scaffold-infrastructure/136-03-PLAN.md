# Plan 03: verify:generate Command & End-to-End Idempotency

## Objective

Implement the `verify:generate --phase <N>` CLI command that reads phase success criteria from ROADMAP.md, test result data, and requirement completion status to produce a pre-filled VERIFICATION.md scaffold. Judgment sections (gap assessment, human verification notes, result determination) are marked for LLM completion. Confirm end-to-end idempotency for both scaffold generators.

## Tasks

### Task 1: Implement verify:generate command and wire routing

- **Type:** auto
- **Files:**
  - `src/commands/scaffold.js` (modify — add cmdVerifyGenerate alongside existing cmdPlanGenerate)
  - `src/router.js` (modify — add verify:generate route)

- **Action:**

  1. **Add `cmdVerifyGenerate(cwd, args, raw)` to `src/commands/scaffold.js`:**

     **Argument parsing:**
     - `--phase <N>` (required) — phase number

     **Data extraction (pre-fill from deterministic sources):**
     - Call `findPhaseInternal(cwd, phaseNum)` to resolve phase directory
     - Parse ROADMAP.md to extract: `goal`, `success_criteria` (numbered list), `requirements` (comma-separated IDs)
     - Parse REQUIREMENTS.md to get requirement descriptions and completion status (checkbox `[x]` vs `[ ]`)
     - Scan phase directory for PLAN.md files — extract `must_haves` (truths, artifacts, key_links) from each plan's frontmatter or body
     - If SUMMARY.md files exist, extract `requirements-completed` fields for completion tracking

     **Scaffold generation with markers:**
     - **Frontmatter** (`<!-- data -->`): `phase`, `name` (phase name), `verified` (ISO timestamp), `status: pending`, `score: '0/N must-haves verified'` (N from count of truths + artifacts + key_links)
     - **Goal Achievement section:**
       - **Observable Truths table** (`<!-- data -->`): Pre-fill rows from success criteria in ROADMAP.md. Each criterion becomes a row: `| # | Truth text | pending | |`. If plans have `must_haves.truths`, include those too.
       - **Required Artifacts table** (`<!-- data -->`): Pre-fill from `must_haves.artifacts` across all plans. Each artifact: `| path | provides | pending | |`
       - **Key Link Verification table** (`<!-- data -->`): Pre-fill from `must_haves.key_links` across all plans. Each link: `| from | to | via | pending | |`
     - **Requirements Coverage table** (`<!-- data -->`): Pre-fill from REQUIREMENTS.md. Each requirement: `| SCAF-01 | description | status | |`. Status pulled from REQUIREMENTS.md checkbox or summary `requirements-completed`.
     - **Gaps Summary section** (`<!-- judgment -->`): Empty template with TODO markers for LLM to fill after inspection
     - **Result section** (`<!-- judgment -->`): TODO marker for LLM to determine pass/gaps_found/human_needed

     **Idempotent merge:**
     - Before writing, check if target `{phase}-VERIFICATION.md` exists
     - If it exists, call `mergeScaffold(existing, fresh, VERIFY_JUDGMENT_SECTIONS)`
     - `VERIFY_JUDGMENT_SECTIONS` = `['Gaps Summary', 'Result', 'Anti-Patterns Found', 'Human Verification Required']`
     - Write merged result
     - If file doesn't exist, write fresh scaffold

     **Output:** JSON with `{ created: true, path, phase, criteria_count, requirement_count, sections: { data: N, judgment: N } }`

  2. **Wire routing in `src/router.js`:**
     - In the `case 'verify':` block (line 558), add `else if (subcommand === 'generate')` branch that parses `--phase` from `restArgs` and calls `lazyScaffold().cmdVerifyGenerate(cwd, options, raw)`
     - The `lazyScaffold()` loader should already exist from Plan 02

  3. **Export** `cmdVerifyGenerate` and `VERIFY_JUDGMENT_SECTIONS` from `scaffold.js`.

- **Verify:** `node bin/bgsd-tools.cjs verify:generate --phase 136 --raw` produces JSON output with `created: true`. The generated VERIFICATION.md contains pre-filled success criteria rows and `<!-- data -->` / `<!-- judgment -->` markers.

- **Done:** `verify:generate --phase <N>` command works end-to-end. Produces pre-filled VERIFICATION.md with data/judgment separation. Routed through `verify:generate` in the CLI. Handles data extraction from ROADMAP.md, REQUIREMENTS.md, and plan must-haves.
