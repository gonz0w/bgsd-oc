  <name>Task 1: Implement the canonical MEMORY.md schema, parser, serializer, and prune candidate logic</name>
  <files>src/commands/memory.js</files>
  <action>
Refactor `src/commands/memory.js` so Phase 145 uses `.planning/MEMORY.md` as the canonical memory store while preserving backward compatibility for existing JSON-store memory commands that other phases may still rely on.

Add a structured-memory layer inside this module with these exact rules from CONTEXT.md and RESEARCH.md:

1. Canonical file path: `.planning/MEMORY.md`.
2. Canonical section order: `Active / Recent`, `Project Facts`, `User Preferences`, `Environment Patterns`, `Correction History`.
3. Entry shape: one lead bullet containing stable ID + semantic type tag + text, followed by indented metadata bullets.
4. Required metadata: `Added`, `Updated`.
5. Optional metadata only when present: `Source`, `Keep`, `Status`, `Expires`, `Replaces`.
6. Correction history stores recurring corrections only; do not introduce autonomous summarization, embeddings, retrieval, or live-use counters.

Implement helpers that:
- bootstrap a default MEMORY.md when missing;
- parse only the fixed schema above (do NOT add a markdown AST dependency);
- serialize back to deterministic markdown with stable section ordering and stable IDs like `MEM-001`;
- preserve supported hand-edited values rather than rewriting everything into a rigid record format;
- compute prune candidates using age + relevance rules: protect `Active / Recent`, protect `Keep: always` (or equivalent pinned marker chosen in implementation), and emit candidate reasons such as `old`, `inactive`, `superseded`, or `duplicate`.

Keep this logic path-agnostic, Node built-in only, and compatible with the single-file CLI bundle.
  </action>
  <verify>Run `npm test -- tests/memory.test.cjs` and confirm the structured memory parser/serializer cases pass.</verify>
  <done>
`src/commands/memory.js` can create, parse, update, and re-serialize `.planning/MEMORY.md` with the five required sections, stable IDs, deterministic metadata ordering, and preview-ready prune candidate reasoning without adding runtime dependencies.
  </done>
