<task type="auto">
  <name>Task 2: Add central routing, mechanical fix validation, and severity-led output shaping</name>
  <files>src/lib/review/routing.js, src/lib/review/fixes.js, src/lib/review/scan.js, src/commands/review.js</files>
  <action>
Implement the post-detection routing stage so `review:scan` enforces the locked user decisions instead of letting each rule decide presentation on its own.

Apply these exact behaviors:
- confidence threshold is centralized and configurable, defaulting to 0.8 / 8-of-10;
- confidence scores stay internal and do not appear in the user-facing report;
- AUTO-FIX is allowed only when the finding is explicitly mechanical and the proposed patch validates cleanly;
- ambiguous, behavioral, or failed mechanical fixes downgrade to ASK instead of aborting the scan;
- ASK findings are grouped by theme, but each finding remains independently answerable later in the workflow;
- the scan output includes both routed findings and aggregate buckets for severity, autofixed counts, ask groups, and suppressed counts; and
- a clean review returns quietly rather than producing an overconfident success banner.

Shape formatted output around severity buckets first, then routing details, so BLOCKER/WARNING/INFO remain the user's primary attention model.
  </action>
  <verify>Run `node bin/bgsd-tools.cjs review:scan --pretty` in a fixture repo and confirm severity-led output plus fix-downgrade behavior; then run `npm test -- tests/review.test.cjs`.</verify>
  <done>
  `review:scan` routes findings consistently, applies only validated mechanical fixes silently, degrades failed fixes to ASK, hides numeric confidence from the user, and emits severity-led summaries with themed ASK groups.
  </done>
</task>
