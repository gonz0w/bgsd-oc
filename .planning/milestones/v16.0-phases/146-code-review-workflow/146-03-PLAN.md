Create `workflows/review.md` as the orchestration layer for Phase 146, keeping the architecture CLI-first and agent-second.

The workflow must:
- run `review:scan` first and treat its JSON as the deterministic findings source of truth;
- present silent AUTO-FIX results only as a concise summary, not a detailed interrupting diff;
- batch ASK findings by theme while still collecting per-finding decisions;
- leave unanswered ASK findings unresolved instead of blocking the workflow;
- hand the surviving routed findings to the judgment-stage agent for structural audit then quality assessment, rather than asking the agent to rediscover scanner findings; and
- end with a severity-led structured report that stays quiet on clean output and is reusable by later readiness/reporting work.

Keep confidence scores internal, preserve the staged-first scope language in the user-facing flow, and note any incomplete-scope warning separately from actual findings.
