<feature>
  <name>Advisory readiness contract with terse color-coded board output</name>
  <files>src/lib/review/readiness.js, src/commands/review.js, tests/review-readiness.test.cjs</files>
  <behavior>
`review:readiness` reports a deterministic readiness snapshot without blocking release flow.

Cases:
  - tests/lint/review/security/TODO/changelog check has evidence and passes → `pass`
  - required signal is unavailable or unconfigured → `skip` with explicit reason
  - check actually runs and fails → `fail`
  - pretty output on TTY → terse board-first advisory view with color-coded status cells and secondary reasons
  - non-ready result → advisory output only, no hidden blocking semantics
  </behavior>
  <implementation>
Add one shared readiness helper that gathers evidence, classifies each check into pass/fail/skip, and formats both JSON and terse color-coded TTY output. Route `review:readiness` through the existing review command module and keep the release command namespace scaffolded for later plans.
  </implementation>
</feature>
