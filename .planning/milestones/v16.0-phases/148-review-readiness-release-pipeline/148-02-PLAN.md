<feature>
  <name>Semver inference and hybrid changelog generation</name>
  <files>src/lib/release/bump.js, src/lib/release/changelog.js, tests/release.test.cjs</files>
  <behavior>
Release analysis should recommend a safe version bump and generate readable grouped release notes before any mutation steps run.

Cases:
  - breaking conventional commit since last tag → major bump suggestion
  - feature commits without breaking change → minor bump suggestion
  - fix/docs/chore-only history → patch bump suggestion
  - ambiguous or messy history → conservative patch with advisory reason and manual override support
  - changelog draft generation → grouped entries from summaries plus commit types in readable markdown
  </behavior>
  <implementation>
Build tag-aware history helpers on top of the existing git layer, centralize ambiguity detection and override handling in the bump helper, and create a changelog generator that merges plan-summary notes with conventional commit grouping into the repo's new CHANGELOG baseline.
  </implementation>
</feature>
