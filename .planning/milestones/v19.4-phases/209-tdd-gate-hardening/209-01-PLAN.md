<feature>
  <name>TDD Gate Hardening</name>
  <files>src/commands/misc/recovery.js</files>
  <tdd-targets>
    <red>node bgsd-tools.cjs execute:tdd validate-tdd-plan --plan-file .planning/phases/209-tdd-gate-hardening/209-01-PLAN.md (plan must be valid structure)</red>
    <green>node bgsd-tools.cjs execute:tdd validate-green --test-cmd "node test/fixture/calc.test.cjs" --test-file test/fixture/calc.test.cjs (must pass with test file unmodified)</green>
    <refactor>node bgsd-tools.cjs execute:tdd validate-refactor --test-cmd "node test/fixture/calc.test.cjs" --prev-count 3 (must pass with same test count as GREEN)</refactor>
  </tdd-targets>
  <behavior>
    Planning-time gate (validate-tdd-plan):
    1. Parses --plan-file argument to extract plan path
    2. Reads plan file and extracts frontmatter
    3. For type:tdd plans, validates: type exists, test_file exists, impl_files exists, steps exists, tdd_rationale exists
    4. Validates step sequence: RED before GREEN before REFACTOR (partial sequences like RED->GREEN valid; REFACTOR alone invalid)
    5. Returns structured proof JSON: { valid: boolean, errors: string[] }
    
    Enhanced GREEN gate:
    1. Parses --test-file argument (optional for mtime tracking)
    2. Records test file mtime+size BEFORE running test command
    3. Runs test command via spawnSync
    4. If --test-file provided: compares mtime+size AFTER run
       - Fast path: if mtime AND size unchanged -> testFileUnmodified = true
       - Fallback: if mtime or size changed -> semantic diff (whitespace-insensitive)
         - Normalize: strip trailing whitespace, normalize line endings
         - If content identical after normalization -> testFileUnmodified = true
    5. Returns structured proof JSON with testFileUnmodified and method used
    
    Enhanced REFACTOR gate:
    1. Parses --prev-count argument
    2. Runs test command via spawnSync
    3. Parses test count from output
    4. Verifies test count === prevCount
    5. Returns structured proof JSON with countUnchanged
  </behavior>
  <implementation>
    cmdTdd(cwd, subcommand, parsedArgs, raw) {
      switch(subcommand) {
        case 'validate-tdd-plan':
          return validateTddPlan(cwd, parsedArgs, raw);
        case 'validate-red':
          return validateRed(cwd, parsedArgs, raw);
        case 'validate-green':
          return validateGreen(cwd, parsedArgs, raw);
        case 'validate-refactor':
          return validateRefactor(cwd, parsedArgs, raw);
        // ... existing cases
      }
    }
    
    validateTddPlan: read plan file, extract frontmatter, validate type:tdd structure, return { valid, errors }
    validateGreen: extended with mtime+size fast path + semantic diff fallback
    validateRefactor: extended with test count verification from --prev-count
  </implementation>
</feature>
