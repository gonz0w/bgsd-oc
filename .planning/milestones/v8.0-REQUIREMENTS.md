# Requirements: bGSD Plugin v8.0

**Defined:** 2026-03-01
**Core Value:** Manage and deliver high-quality software with high-quality documentation, while continuously reducing token usage and improving performance

## v8.0 Requirements

Requirements for v8.0 Performance & Agent Architecture milestone. Each maps to roadmap phases.

### Cache Layer

- [x] **CACHE-01**: SQLite read cache module persists parsed data across CLI invocations using `node:sqlite` (DatabaseSync)
- [x] **CACHE-02**: Cache validates freshness via mtime on every read, re-parses on stale
- [x] **CACHE-03**: Any file write through gsd-tools invalidates the corresponding cache entry
- [x] **CACHE-04**: Cache gracefully degrades to in-memory Map on Node <22.5 (no crashes, no warnings)
- [x] **CACHE-05**: User can pre-populate cache via `cache warm` command after checkout or pull
- [x] **CACHE-06**: User can compare performance baselines via `profiler compare` command

### Agent Architecture

- [x] **AGENT-01**: RACI matrix maps every product lifecycle step to exactly one responsible agent
- [x] **AGENT-02**: Agent context loading is manifest-driven — agents receive pre-declared files, no ad-hoc discovery
- [x] **AGENT-03**: Overlapping agents consolidated (integration-checker→verifier, synthesizer→roadmapper), 11→9 agents
- [x] **AGENT-04**: Automated lifecycle audit reports gaps (uncovered steps) and overlaps (multiple owners)
- [x] **AGENT-05**: Agent manifests declare max token budgets; context builder enforces limits

### Command Consolidation

- [x] **CMD-01**: Orphan commands grouped under namespaced parents (~15-20 commands relocated)
- [x] **CMD-02**: All references (tests, workflows, agent prompts) updated to new command names
- [x] **CMD-03**: Milestone wrapup workflow generates documentation artifact automatically

### Performance

- [x] **PERF-01**: Hot-path profiler instrumentation covers file reads, git ops, markdown parsing, and AST analysis
- [x] **PERF-02**: Baseline comparison tool shows before/after timing deltas with regression highlighting

## Out of Scope

Explicitly excluded. Documented to prevent scope creep.

| Feature | Reason |
|---------|--------|
| Full async I/O rewrite | Wrong paradigm for <5s CLI — sync I/O is appropriate |
| Agent self-spawning | Caps at 12 roles by design; intelligence = data, not agents |
| better-sqlite3 native addon | Breaks single-file deploy; node:sqlite preferred |
| Commander.js/yargs migration | Adds ~200KB dep for marginal benefit; existing router sufficient |
| Command alias backward compat | Single user — rename directly, update all references |
| Cross-invocation daemon mode | Requires long-running process; SQLite cache achieves same goal |
| Real-time file watching | CLI is short-lived; mtime check on read is sufficient |
