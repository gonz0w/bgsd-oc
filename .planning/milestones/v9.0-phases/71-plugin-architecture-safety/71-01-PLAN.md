<task type="auto">
  <name>Task 2: Implement safeHook error boundary with retry, timeout, circuit breaker, and logging</name>
  <files>src/plugin/safe-hook.js</files>
  <action>
Create `src/plugin/safe-hook.js` implementing the universal error boundary per CONTEXT.md decisions:

1. `safeHook(name, fn, options)` function:
   - `name`: string identifier for the hook (used in logs and toasts)
   - `fn`: the actual hook function `(input, output) => Promise<void>`
   - `options`: `{ timeout = 5000 }` — async hook timeout in ms
   - Returns a wrapped function with the same signature

2. Error boundary behavior (exact order per CONTEXT.md):
   a. **Disabled check**: If circuit breaker tripped (`disabled === true`), return immediately (skip hook)
   b. **Debug bypass**: If `process.env.BGSD_DEBUG === '1'`, call `fn(input, output)` directly with NO wrapping — exceptions propagate for debugging
   c. **Generate correlation ID**: 8-char hex via `crypto.randomBytes(4).toString('hex')`
   d. **Start timer**: `Date.now()` for slow hook logging
   e. **Retry loop** (2 attempts):
      - Attempt 1: call `fn` with timeout race. On error: continue to attempt 2 (silent retry)
      - Attempt 2: call `fn` with timeout race. On error: log and potentially disable
   f. **Timeout**: `Promise.race([fn(input, output), timeout_promise])` — 5-second default
   g. **On success** (either attempt):
      - Reset `consecutiveFailures` to 0
      - Check elapsed time: if > 500ms, log slow hook warning via logger
      - Return (hook succeeded)
   h. **On final failure** (both attempts failed):
      - Increment `consecutiveFailures`
      - Log error to file AND stderr via logger: include hook name, error message, stack trace, correlation ID
      - Show toast: `console.log("[bGSD] Hook failed: ${name} [${correlationId}]")`
      - If `consecutiveFailures >= 3`: set `disabled = true`, show toast: `console.log("[bGSD] Hook ${name} disabled after repeated failures")`

3. State is per-hook instance (closure variables):
   - `consecutiveFailures` = 0
   - `disabled` = false
   - Both reset implicitly when plugin reloads (new process = fresh closure)

4. Exports: `module.exports = { safeHook }`

5. Dependencies: Import `createLogger` from `./logger.js`. Initialize logger lazily on first error (avoid creating log file if no errors occur). Use `os.homedir()` to resolve log directory: `path.join(homedir(), '.config', 'opencode')` — NOT per-project, global.

Note: This is ONE universal `safeHook` wrapper (agent's discretion decision) — not type-specific wrappers. The hook signature is uniform across all OpenCode hooks.
  </action>
