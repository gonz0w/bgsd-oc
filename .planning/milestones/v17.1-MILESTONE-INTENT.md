# Milestone Intent: v17.1 Workflow Reliability & Foundation Hardening

**Archived:** 2026-03-30

## Why Now

The repo has enough recurring execution and verification failures in `.planning/memory/lessons.json` that workflow friction is no longer isolated cleanup. This milestone turns those repeated failures, plus the codebase efficiency and reliability audit, into focused hardening work so future milestones can execute faster and with less drift.

## Targeted Outcomes

- Improve JJ-first workflow execution reliability in real detached or dirty workspace conditions
- Reduce avoidable verification false negatives and stale-command retries during planning and execution
- Strengthen shared planning contracts so state, summaries, and workflow guidance stay deterministic as parallel execution grows
- Advance the enduring project objective by reducing token waste and operator friction in the planning/execution loop

## Priorities

- Fix the highest-frequency lessons first, especially JJ commit-path failures and verifier metadata gaps
- Prefer shared helper and contract fixes over one-off prompt wording repairs
- Keep changes measurable through regression coverage, readback validation, and hotspot evidence
- Preserve backward compatibility for planning artifacts and user-facing workflows

## Non-Goals

- Changing how lessons are captured or managed
- Broad architecture rewrites without milestone-scoped acceptance criteria
- New product features unrelated to workflow hardening
- Speculative optimization without repeated-work or reliability evidence

## Notes

- Use `.planning/memory/lessons.json` and `.planning/research/CODEBASE-EFFICIENCY-RELIABILITY-AUDIT-PRD.md` as the primary planning seeds.
- Treat workflow guidance changes as incomplete unless the underlying helper/runtime path is also fixed or explicitly bounded.
