<task type="auto">
  <name>Task 1: Audit COMMAND_HELP for stale entries and CONFIG_SCHEMA for unused keys</name>
  <files>src/lib/constants.js</files>
  <action>
    **Step 1: Cross-reference COMMAND_HELP keys against the router.**

    Load the router command table from `src/router.js` and extract every registered command string. Then check each key in `COMMAND_HELP` (in `src/lib/constants.js`) — if a COMMAND_HELP key doesn't match any router command, it's stale.

    Note: COMMAND_HELP has both namespaced (`research:nlm-ask`) and space-form (`research nlm-ask`) entries. The router accepts both forms. Only remove entries where NEITHER form exists in the router.

    **Step 2: Check for duplicate help text.**

    Many commands have identical help text in both `command:subcommand` and `command subcommand` forms. Where duplicates exist, keep only the space-form (canonical) and remove the colon-form duplicate. The router dispatches both to the same handler, but COMMAND_HELP only needs one entry — the help command resolves both forms to the same key.

    Actually, before removing colon-form duplicates, check how the help lookup works in the codebase. If help lookup uses the exact key match, both forms may need to stay. Read the help command implementation first. If it normalizes colon→space before lookup, then colon-form duplicates are safe to remove.

    **Step 3: Audit CONFIG_SCHEMA keys.**

    For each key in CONFIG_SCHEMA, grep the codebase (excluding constants.js itself) to verify it's referenced somewhere — either in config-get/config-set handling, init output, or documentation. Flag any keys with zero references.

    **Step 4: Remove stale entries.**

    Delete any COMMAND_HELP entries for non-existent commands. Delete any CONFIG_SCHEMA keys with zero references. If duplicate colon-forms are safe to remove (per Step 2 analysis), remove them.

    Run `npm run build` after changes.
  </action>
  <verify>

    `node -e "const {COMMAND_HELP} = require('./src/lib/constants.js'); const keys = Object.keys(COMMAND_HELP); console.log('Help entries:', keys.length)"` — count should be lower than before.
    `npm test` — all tests pass.
  </verify>
  <done>All COMMAND_HELP entries correspond to real router commands. No CONFIG_SCHEMA keys are orphaned. Duplicate help text consolidated where safe. Tests pass.</done>
