  <action>
Analyze all 9 agent .md files in agents/ and all orchestrator workflows in workflows/ (plan-phase.md, execute-phase.md, execute-plan.md, new-project.md, new-milestone.md, verify-work.md, audit-milestone.md, map-codebase.md, discuss-phase.md, research-phase.md, debug — the key orchestrators) to map which agent is responsible for which lifecycle step.

Create references/RACI.md with:

**1. Lifecycle Steps (15-25 at sub-stage level per CONTEXT.md decision):**
Enumerate steps like: project-init, intent-capture, milestone-setup, research-synthesis, phase-discussion, phase-research, plan-creation, plan-checking, plan-approval, execution-dispatch, task-execution, commit-management, checkpoint-handling, deviation-handling, phase-verification, milestone-audit, codebase-mapping, debug-investigation, progress-reporting, state-management, etc.

**2. RACI Matrix Table:**
Format: `| Step | Responsible (R) | Accountable (A) | Consulted (C) | Informed (I) |`
- Each step gets exactly ONE Responsible agent (the agent that does the work)
- Orchestrator commands (the user's slash commands) appear as Accountable (they spawn the R agent)
- Multiple agents can share A/C/I columns without triggering warnings
- Include the reviewer agent (references/reviewer-agent.md) in the matrix for evaluation
- "User" appears where the human is R or A (e.g., phase-discussion decisions)

**3. Handoff Contracts Section:**
For each agent-to-agent transition, document:
- Source agent → Target agent
- Artifact passed (file path)
- Required sections in that artifact (section-level detail per CONTEXT.md decision)
- Example: "gsd-phase-researcher → gsd-planner: RESEARCH.md must contain ## Findings, ## Risks, ## Recommendations"
- Example: "gsd-planner → gsd-executor: PLAN.md must contain frontmatter (phase, plan, type, wave, depends_on, files_modified, autonomous, must_haves), <objective>, <tasks> with <name>, <files>, <action>, <verify>, <done>"
- Example: "gsd-executor → gsd-verifier: SUMMARY.md must contain ## What Was Built, ## Files Modified, ## Verification Results"

Key transitions to document:
- project-researcher → roadmapper (RESEARCH files → roadmap creation)
- roadmapper → planner (ROADMAP.md → phase planning)
- phase-researcher → planner (RESEARCH.md → plan creation)
- planner → plan-checker (PLAN.md → plan verification)
- plan-checker → planner (checker issues → plan revision)
- planner → executor (PLAN.md → task execution)
- executor → verifier (SUMMARY.md → phase verification)
- codebase-mapper → planner (codebase/*.md → planning context)
- debugger operates standalone (spawned by user, no handoff chain)

Note: The reviewer-agent (references/reviewer-agent.md) should be included in the RACI matrix. Its disposition (merge, deploy standalone, or remove) will be determined in Plan 02 based on overlap analysis.
