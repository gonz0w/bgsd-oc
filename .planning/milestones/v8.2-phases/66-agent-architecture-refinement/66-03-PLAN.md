**1. Update `cmdAgentAudit` to handle 15-25 lifecycle steps:**

The current audit hardcodes 7 lifecycle steps: `['Init', 'Discuss', 'Research', 'Plan', 'Execute', 'Verify', 'Complete']`. Update to:
- Read lifecycle steps dynamically FROM the RACI.md table (parse the Step column) instead of hardcoding
- Keep the hardcoded list as fallback if RACI.md parsing fails
- Update `parseRaciMatrix` to handle the new sub-stage step names (e.g., "plan-creation", "plan-checking" with hyphens, not just single words)
- The current regex `match(/^\|\s*(\w+)\s*\|\s*([^|]+?)\s*\|/)` won't match hyphenated step names — fix to: `match(/^\|\s*([\w-]+)\s*\|\s*([^|]+?)\s*\|/)`
- Update the overlap detection: per CONTEXT.md "permissive" rule, only flag when TWO agents have R on the SAME step. Multiple A/C/I is fine.

**2. Add `cmdAgentValidateContracts` function:**

New subcommand: `gsd-tools util:agent validate-contracts [--phase <N>]`

Logic:
1. Read all agent .md files, extract `inputs` and `outputs` from frontmatter
2. For each agent's `outputs`, check if the declared file exists in the expected location:
   - If `--phase <N>` provided, check `.planning/phases/<N>-*/` for phase-specific outputs (PLAN.md, SUMMARY.md, RESEARCH.md, VERIFICATION.md)
   - Without `--phase`, validate contract DECLARATIONS only (check that inputs/outputs are well-formed)
3. For each output file that exists, check required_sections:
   - Parse the file content and verify each section heading listed in `required_sections` is present
   - Missing sections = ERROR (per CONTEXT.md: "produces errors, not just warnings")
4. Output JSON:
```json
{
  "agents_checked": 9,
  "contracts_valid": 7,
  "contracts_invalid": 2,
  "errors": [
    { "agent": "gsd-phase-researcher", "output": "RESEARCH.md", "missing_sections": ["## Risks"] }
  ],
  "status": "fail"
}
```

**3. Add router entry:**

In src/router.js, find the `util:agent` dispatch section and add:

  lazyAgent().cmdAgentValidateContracts(cwd, raw, args);
