<task type="auto">
  <name>Task 1: Create dead code audit script with router cross-reference</name>
  <files>audit-exports.js, .planning/baselines/audit/dead-code-report.json</files>
  <action>
Create `audit-exports.js` (Node.js script, zero dependencies) that:

1. **Parse router dispatch table** from `src/router.js`:
   - Extract all `lazyXxx().funcName` calls using regex `/lazy\w+\(\)\.(\w+)/g`
   - Build a Set of all router-dispatched function names (~158 functions)

2. **Parse cross-module imports** from `src/commands/*.js` and `src/lib/*.js`:
   - For each source file, find `require('...')` calls to sibling modules
   - Extract which named imports are destructured (e.g., `const { parseVtt } = require('./research')`)
   - Build a Map of `exportName → [importing files]` for cross-module consumers

3. **Run knip** via `execSync('npx knip --include exports,files --reporter json', { encoding: 'utf-8' })`:
   - Parse JSON output: `files` array (unused files) and `issues` object (unused exports grouped by file)
   - Each issue has `{ file, exports: [{ name, line, col }] }`

4. **Classify each knip-reported unused export** into categories:
   - `router_consumed`: export name appears in router dispatch Set → FALSE POSITIVE from knip
   - `cross_module`: export name appears in cross-module import Map → FALSE POSITIVE from knip
   - `truly_dead`: export name NOT in router dispatch AND NOT in cross-module imports → CANDIDATE for removal
   - `internal_helper`: exported but only used within same file (check with simple regex) → review candidate

5. **Write structured report** to `.planning/baselines/audit/dead-code-report.json`:
   ```json
   {
     "generated_at": "ISO timestamp",
     "knip_total_unused": 263,
     "unused_files": ["src/lib/review/stage-review.js"],
     "classification": {
       "router_consumed": [{ "name": "cmdStateLoad", "file": "src/commands/state.js", "line": 702 }],
       "cross_module": [{ "name": "parseVtt", "file": "src/commands/research.js", "consumers": ["src/commands/features.js"] }],
       "truly_dead": [{ "name": "searchPhaseInDir", "file": "src/lib/helpers.js", "line": 1015 }],
       "internal_helper": []
     },
     "summary": {
       "router_consumed_count": N,
       "cross_module_count": N,
       "truly_dead_count": N,
       "unused_files_count": 1
     }
   }
   ```

6. **Print summary to stdout** (TTY-friendly):
   ```
   Dead Code Audit Report
   ─────────────────────
   Knip total: 263 unused exports, 1 unused file
   Router-consumed (false positives): N
   Cross-module (false positives): N
   Truly dead: N
   Unused files: 1
   Report: .planning/baselines/audit/dead-code-report.json
   ```

Follow project conventions: `'use strict'`, 2-space indent, single quotes, `execFileSync` for subprocess calls (but `execSync` is OK for npx since it needs shell). Use `fs.mkdirSync` with recursive for output dir.
  </action>
  <verify>
Run `node audit-exports.js` — should complete without errors, print summary, and write `.planning/baselines/audit/dead-code-report.json`. Verify JSON is valid: `node -e "JSON.parse(require('fs').readFileSync('.planning/baselines/audit/dead-code-report.json','utf-8'))"`. Verify `truly_dead` array has fewer entries than knip's 263 total (proving router cross-reference works).
  </verify>
  <done>dead-code-report.json exists with classified exports. Router-consumed exports are separated from truly dead ones. Summary printed to console.</done>
