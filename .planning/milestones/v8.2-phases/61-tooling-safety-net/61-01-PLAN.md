<task type="auto">
  <name>Task 1: Enable esbuild metafile and per-module build analysis</name>
  <files>build.js, .planning/baselines/build-analysis.json</files>
  <action>
  Modify build.js to enable esbuild metafile output and produce per-module byte attribution:

  1. Add `metafile: true` to the esbuild.build() options object.

  2. Capture the build result: `const result = await esbuild.build({...})` (currently the return value is discarded).

  3. After the build completes (after existing smoke test and bundle size tracking), add a metafile analysis section:
     - Parse `result.metafile.outputs['bin/gsd-tools.cjs'].inputs` — this is an object keyed by source file path with `{ bytesInOutput }` values.
     - Group files by directory prefix (e.g., `src/lib/`, `src/commands/`, `src/`, `node_modules/acorn/`, `node_modules/tokenx/`).
     - For each group: compute subtotal bytes, list individual files with their byte counts.
     - Console output format:
       ```
       Module analysis:
         src/lib/          482KB (15 files)
           constants.js     52KB
           helpers.js       31KB
           ...
         src/commands/     312KB (18 files)
           ...
         node_modules/     180KB (2 packages)
           ...
       ```
     - Sort groups by total size descending, files within groups by size descending.
     - Warn (console.warn, don't exit) if any single source file exceeds 50KB in the output.

  4. Write the full per-module breakdown to `.planning/baselines/build-analysis.json`:
     ```json
     {
       "timestamp": "ISO string",
       "bundle_size_kb": 1216,
       "modules": {
         "src/lib/constants.js": { "bytes": 53248, "kb": 52 },
         "src/lib/helpers.js": { "bytes": 31744, "kb": 31 },
         ...
       },
       "groups": {
         "src/lib/": { "bytes": 493568, "kb": 482, "file_count": 15 },
         "src/commands/": { "bytes": 319488, "kb": 312, "file_count": 18 },
         ...
       }
     }
     ```

  5. Also write the raw esbuild metafile to `/tmp/gsd-metafile.json` for ad-hoc analysis (use `fs.writeFileSync` with `JSON.stringify(result.metafile, null, 2)`).

  Keep all existing build functionality intact (shebang plugin, smoke test, bundle size tracking, budget gate).
  </action>
  <verify>
  Run `npm run build` and confirm:
  - Build succeeds with same bundle size as before (~1216KB)
  - Console output includes "Module analysis:" section with per-directory groups
  - `.planning/baselines/build-analysis.json` exists and contains modules + groups keys
  - `/tmp/gsd-metafile.json` exists
  - All existing smoke test and budget checks still pass
  </verify>
  <done>npm run build produces per-module byte attribution in console and persists build-analysis.json with per-file and per-directory breakdowns</done>
</task>

  4. Verify madge runs against src/:
