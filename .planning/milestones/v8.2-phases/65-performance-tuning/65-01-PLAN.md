<task type="auto">
  <name>Task 1: Lazy-load acorn in ast.js</name>
  <files>src/lib/ast.js</files>
  <action>
Move `const acorn = require('acorn')` from line 5 (top-level) to inline `require('acorn')` inside each function that uses it. There are 4 acorn consumer functions in ast.js:

1. `extractSignatures()` — uses `acorn.parse()`
2. `extractExports()` — uses `acorn.parse()`
3. `computeComplexity()` — uses `acorn.parse()`
4. `generateRepoMap()` — uses `acorn.parse()`

Pattern: At the top of each function body, add `const acorn = require('acorn');` (Node.js caches require calls, so subsequent calls in the same process are free).

Remove the top-level `const acorn = require('acorn');` on line 5.

This means non-AST commands never load/parse acorn's 230KB, improving cold-start for all other commands.

Note: `src/commands/codebase.js` already lazy-loads ast.js itself via inline `require('../lib/ast')` (4 call sites, lines 1269, 1351, 1424, 1451). So the full chain is now: router → lazy codebase.js → lazy ast.js → lazy acorn. Three levels of lazy-loading, but only for the 4 commands that need AST.
  </action>
  <verify>
1. `npm run build` succeeds and reports bundle size
2. `node bin/gsd-tools.cjs state get` works (non-AST command — should NOT load acorn)
3. `GSD_PROFILE=1 node bin/gsd-tools.cjs state get 2>&1` shows no acorn-related timing entries
4. `npm test` — all tests pass
  </verify>
  <done>Top-level `require('acorn')` removed from ast.js. Acorn only loaded when AST functions are called. Bundle builds successfully.</done>
</task>
