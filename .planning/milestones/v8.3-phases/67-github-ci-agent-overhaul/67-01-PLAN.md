**2. Add `<deviation_rules>` block (after `</execution_flow>`, before `<completion_format>`):**
- Create CI-specific deviation rules (NOT executor's code-change rules). Structure:
  - **RULE 1: Auto-fix simple true positives** — Trigger: low-complexity scanning alert. Examples: unused imports, missing input sanitization (simple), hardcoded test credentials. Action: fix, commit, repush, track.
  - **RULE 2: Auto-fix build/lint/test failures** — Trigger: non-scanning check failure. Examples: TypeScript error, ESLint violation, failing test from code change. Action: read error output, fix, commit, repush, track.
  - **RULE 3: Dismiss false positives (low severity)** — Trigger: note/warning severity alert clearly false positive. Examples: alert in test file, pattern match on variable name, vendored code. Action: dismiss via API with reasoning, track as decision.
  - **RULE 4: Escalate to user** — Trigger: medium+ severity suspected false positive, or complex fix requiring architectural changes. Examples: alert requiring new DB table, library replacement, ambiguous security finding. Action: STOP, return CHECKPOINT REACHED.
  - **RULE PRIORITY:** Rule 4 (stop) > Rule 3 (dismiss) > Rules 1-2 (fix) > genuinely unsure = Rule 4
  - **FIX ATTEMPT LIMIT:** After `{MAX_FIX_ITERATIONS}` attempts, return checkpoint with remaining issues.
  - **SCOPE BOUNDARY:** Only fix issues reported by CI checks. Do not proactively fix pre-existing issues in touched files.
  - **Config overrides:** Check `config.json` for project-specific deviation rules via `node $GSD_HOME/bin/gsd-tools.cjs util:config-get ci.deviation_rules 2>/dev/null`

**3. Add `<state_ownership>` block (after `</deviation_rules>`):**
- ~15 lines explaining: CI agent updates STATE.md when invoked directly (not spawned by parent workflow).
- Detection mechanism: If prompt contains `<spawned_by>` tag, skip state updates — return data in CI COMPLETE for parent to record.
- When invoked directly: use gsd-tools commands for decisions and session info.

**4. Trim `<step name="analyze_failures">` to reference deviation_rules:**
