**Migration pattern for EACH agent:**

1. **Add `<skills>` section** — Insert after frontmatter/role block, before execution flow. Use a table format:
```markdown
<skills>
| Skill | Provides | When to Load | Placeholders |
|-------|----------|--------------|--------------|
| project-context | Project discovery block | Always (eager) | action="{{action_verb}}" |
| commit-protocol | Task commit format | During task completion | phase="{{PHASE}}", plan="{{PLAN}}" |
| ... | ... | ... | ... |
</skills>
```

2. **Replace inline blocks with `<skill:name />` tags** — For each extractable block, remove the full content and replace with an inline skill tag at the point where it's referenced in the execution flow:
   - `<project_context>...</project_context>` → `<skill:project-context action="..." />`
   - `<structured_returns>...</structured_returns>` → `<skill:structured-returns section="agent-name" />`
   - `<goal_backward>...</goal_backward>` → `<skill:goal-backward />`
   - etc.

3. **KEEP these blocks** (do NOT extract):
   - Role/identity block (who the agent is, CRITICAL rules)
   - PATH SETUP block (bootstrap — must run before tool calls)
   - Mandatory Initial Read block (bootstrap — must run before tool calls)
   - `<execution_flow>` (the agent's step-by-step process — this stays, with inline skill tags added)
   - Any unique conditional logic, decision trees specific to this agent's flow

4. **No migration trail comments** — Don't leave `<!-- Moved to skill -->` comments. Clean removal.
