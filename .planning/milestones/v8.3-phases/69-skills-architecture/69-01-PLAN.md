  <name>Task 1: Extend build.cjs with skill manifest collection, validation, and index generation</name>
  <files>build.cjs</files>
  <action>
Add three new capabilities to build.cjs:

**1. Manifest collection** — After the existing `collectFiles('references', ...)` call (~line 218), add:
```javascript
collectFiles('skills', () => true);
```
This ensures all files in skills/ subdirectories (SKILL.md + supporting files) are included in bin/manifest.json.

**2. Skill validation** — Add a `validateSkills()` function called after manifest generation. It must:
- Scan `skills/` for subdirectories (each is a skill)
- Skip `skill-index` directory (auto-generated)
- For each skill directory: verify SKILL.md exists, parse YAML frontmatter, check `name` and `description` fields present
- Parse all `<skill:name />` cross-reference tags in SKILL.md content and verify each referenced skill exists as a directory
- Collect errors and fail the build (`process.exit(1)`) if any validation errors found
- Log success count on pass: `Skills validated: N skills, 0 errors`

Use the RESEARCH.md code example as a starting point but improve: also validate that `<!-- section: X -->` markers exist for any `sections` listed in frontmatter.

**3. Skill index generation** — Add a `generateSkillIndex()` function called after validation. It must:
- Scan all skill directories (excluding `skill-index` itself)
- Read each SKILL.md's frontmatter (name, description, type, agents)
- Generate `skills/skill-index/SKILL.md` with YAML frontmatter and a markdown table of all skills
- Create the `skill-index` directory if it doesn't exist
- Use simple regex-based frontmatter parsing (no YAML library — zero dependencies rule)

Call order: manifest collection → `validateSkills()` → `generateSkillIndex()` → write manifest.json

Note: If skills/ directory doesn't exist yet (e.g., before Plan 02 runs), all three functions should silently skip.

2. Add skills case to `dest_for_file()` function — insert before the `*) echo "$DEST/$file" ;;` catch-all:

   ```bash
   skills/*) echo "$SKILL_DIR/${file#skills/}" ;;
   ```
   This preserves subdirectory structure (e.g., `skills/commit-protocol/SKILL.md` → `~/.config/opencode/skills/commit-protocol/SKILL.md`).

3. Add `$SKILL_DIR` to the `mkdir -p` command (~line 65):
   ```bash

   ```

4. Add skills directory to placeholder substitution (~after line 132):
   ```bash
   find "$SKILL_DIR" -name '*.md' -exec sed -i "s|__OPENCODE_CONFIG__|$OPENCODE_CFG|g" {} + 2>/dev/null || true
   ```
   The `2>/dev/null || true` handles the case where no .md files exist yet.

5. Add skill reference validation after manifest sync and before smoke test. After the stale files cleanup section (~line 120), add:
   ```bash
   # Validate agent skill references
   echo "Validating skill references..."
   SKILL_ERRORS=0
   for agent in "$AGENT_DIR"/gsd-*.md; do
     [ -f "$agent" ] || continue
     REFS=$(grep -oP '<skill:([a-z0-9-]+)' "$agent" 2>/dev/null | sed 's/<skill://' | sort -u)
     for ref in $REFS; do
       if [ ! -f "$SKILL_DIR/$ref/SKILL.md" ]; then
         echo "  ⚠ $(basename "$agent"): references missing skill '$ref'"
         SKILL_ERRORS=$((SKILL_ERRORS + 1))
       fi
     done
   done
   if [ $SKILL_ERRORS -gt 0 ]; then
     echo "  ⚠ $SKILL_ERRORS broken skill references (non-fatal warning)"
   else
     echo "  ✅ All skill references valid"
   fi
   ```
   Note: Make this a WARNING not a fatal error — during migration, agents may reference skills before they're created in the same deploy cycle. Once migration is complete, this becomes a safety net.

6. Add skill count to the summary output (~line 150):
   ```bash
   SKILL_COUNT=$(find "$SKILL_DIR" -maxdepth 2 -name 'SKILL.md' 2>/dev/null | wc -l)
   echo "  Skills deployed: $SKILL_COUNT"
   ```
