**Description tuning (SKIL-04):**

Review every SKILL.md description field and verify:

1. **Uniqueness** — No two descriptions are identical or nearly identical
   ```bash
   grep -h "^description:" skills/*/SKILL.md | sort | uniq -c | sort -rn
   ```
   Any count > 1 means duplicate descriptions that need differentiation.

2. **Specificity** — Each description clearly identifies what the skill provides, so an agent searching for "how to commit code" would find `commit-protocol` and NOT `git-integration` or `state-update-protocol`
   
   Review each description against this checklist:
   - Does it mention the specific domain? (commits, checkpoints, verification, etc.)
   - Does it mention who uses it? (executor, planner, all agents)
   - Could it be confused with another skill? If yes, make it more specific

3. **Accuracy** — Description matches actual content
   - Read each SKILL.md content and verify the description accurately represents it
   - Adjust descriptions that are too broad or too narrow

**False positive/negative test:**

For each agent, verify its `<skills>` table lists exactly the skills it needs — no extra, no missing:

1. Read each agent's `<skills>` table
2. Read each agent's `<execution_flow>` for inline `<skill:name />` tags
3. Verify every inline tag appears in the skills table
4. Verify no skills table entry is never referenced inline (unused skill = false positive)

Fix any mismatches found.
