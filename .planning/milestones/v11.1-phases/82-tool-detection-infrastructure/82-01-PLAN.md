4. **getToolStatus() function**:
   - Returns status for all supported tools
   - Returns { toolName: { available, path, name, description } }

NOTE: Use execFileSync NOT execSync to prevent shell injection. Use array args: execFileSync('which', [binaryName]).

<task type="auto">
  <name>Task 3: Create graceful fallback wrapper module</name>
  <files>src/lib/cli-tools/fallback.js</files>
  <action>
Create `src/lib/cli-tools/fallback.js` with:

1. **withToolFallback(toolName, cliFn, nodeJsFallback) function**:
   - First checks if tool is available via detectTool()
   - If available: executes cliFn() and returns { success: true, usedFallback: false, result }
   - If unavailable: executes nodeJsFallback() and returns { success: true, usedFallback: true, guidance, result }
   - On error: returns { success: false, error }

2. **OperationResult structure**:
   ```javascript
   {
     success: boolean,       // Operation succeeded (regardless of fallback)
     usedFallback: boolean, // Whether Node.js fallback was used
     guidance?: {           // Present if tool unavailable
       name: string,
       description: string,
       installCommand: string,
       installInstructions: string
     },
     result?: any,          // Result from cliFn or nodeJsFallback
     error?: string         // Present if operation failed
   }
   ```

3. **Usage pattern**:
   ```javascript
   const result = withToolFallback(
     'ripgrep',
     () => execFileSync('rg', ['pattern', path]),
     () => fallbackGrep(pattern, path)
   );
   
   if (result.usedFallback) {
     console.log('Note: Using Node.js fallback. ' + result.guidance.installCommand);
   }
   ```

4. **Ensure fallback mention is per-operation**: Each calling operation decides whether to display guidance
  </action>

"</parameter>
<done>
fallback.js exports withToolFallback() function that wraps CLI operations with Node.js fallback, returns structured result with guidance when fallback used</done>
