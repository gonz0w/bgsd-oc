Create src/lib/reports/milestone-summary.js with:
- generateMilestoneSummary(milestoneId, options) - Main function to generate summary
  - Accepts milestone identifier (version like "v11.0") or null for current
  - options: { format: 'console' | 'json', save: boolean, filePath: string }

- formatMilestoneReport(data, format) - Format data for display
  - data includes: name, dates (started, completed), phasesCompleted, totalPhases, tasksCompleted, totalTasks, timeMetrics, qualityScores, velocityMetrics
  - format: 'console' uses format.js tables, 'json' returns raw data
- saveMilestoneReport(content, filePath) - Save report to file
  - Default location: `.planning/milestones/{milestone-version}/summary.md`

Data sources to read:
- .planning/ROADMAP.md - phases, requirements
- .planning/milestones/{version}-ROADMAP.md - archived milestone details
- .planning/STATE.md - current metrics
- .planning/phases/*/phase-*-SUMMARY.md files - task completion data

Report sections:
1. Header: Milestone name, version, dates
2. Overview: Phases completed/total, tasks completed/total
3. Timeline: Started date, completed date, duration
4. Quality: Quality scores if available (from phase summaries)
5. Velocity: Tasks per session, trend (sparkline from VIS-05)
6. Phases table: Phase name, plans, tasks, status

Use format.js for table rendering (formatTable), section headers, and box rendering.
