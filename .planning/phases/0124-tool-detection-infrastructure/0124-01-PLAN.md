  <action>
Upgrade `src/lib/cli-tools/detector.js` with three capabilities:

**1. File-based cache (cross-invocation persistence)**
- Add a `loadFileCache()` function that reads `.planning/.cache/tools.json` if it exists and is <5 minutes old.
- Add a `saveFileCache()` function that writes current detection results to `.planning/.cache/tools.json`.
- On `detectTool()` call: check in-memory Map first → check file cache second → detect from PATH last.
- After fresh detection, write to both in-memory Map AND file cache.
- Cache file format: `{ "timestamp": <epoch_ms>, "results": { "ripgrep": { available, path, version, name, description }, ... } }`.
- Use `fs.mkdirSync` with `recursive: true` to create `.planning/.cache/` if needed.
- Wrap all file I/O in try/catch — silent failure, fall back to in-memory-only if file operations fail.
- Export `CACHE_TTL_MS` for test access.
- Add a `setCachePath(path)` function for testing (overrides default `.planning/.cache/tools.json`).

**2. Cross-platform PATH resolution**
- Replace `execFileSync('which', [binaryName])` with a platform-aware function `resolveToolPath(binaryName)`:
  - On `process.platform === 'win32'`: use `execFileSync('where.exe', [binaryName])` and take the first line of output.
  - On all other platforms: use `execFileSync('which', [binaryName])`.
  - Both wrapped in try/catch returning `null` on failure.

**3. Version comparison**
- Add `parseVersion(versionString)` that extracts semver from version output strings. Handle common formats:
  - `"ripgrep 15.1.0"` → `{ major: 15, minor: 1, patch: 0 }`
  - `"jq-1.8.1"` → `{ major: 1, minor: 8, patch: 1 }`
  - `"fd 10.4.2"` → `{ major: 10, minor: 4, patch: 2 }`
  - `"gh version 2.88.0 (2026-03-11)"` → `{ major: 2, minor: 88, patch: 0 }`
  - `"yq 3.4.3"` → `{ major: 3, minor: 4, patch: 3 }`
  - Use regex: `/(\d+)\.(\d+)\.(\d+)/` to extract first semver match.
  - Return `null` if no version found.
- Add `meetsMinVersion(toolName, minVersionStr)` that:
  - Calls `detectTool(toolName)` to get current version.
  - Parses both current and minimum version.
  - Returns `{ meets: boolean, current: string|null, required: string }`.
- Export: `parseVersion`, `meetsMinVersion`, `resolveToolPath`, `setCachePath`, `CACHE_TTL_MS`.

Keep all existing exports (`TOOLS`, `detectTool`, `getToolStatus`, `clearCache`) working with the same signatures. The `clearCache()` function should also clear the file cache (delete the file).

**2. Add `cmdDetectTools` to src/commands/tools.js**

Create a new `cmdDetectTools(cwd, raw)` function that outputs the flat JSON array format from CONTEXT.md:

```javascript
function cmdDetectTools(cwd, raw) {
  const status = getToolStatus();
  const result = Object.entries(status).map(([key, info]) => ({
    name: info.name,
    cmd: TOOLS[key]?.aliases?.[0] || key,
    available: info.available,
    ...(info.version ? { version: info.version } : {}),
    ...(info.path ? { path: info.path } : {}),
    ...(!info.available ? { install: getInstallCommand(key) } : {})
  }));
  output(result, raw);
}
```

Key design:
- `cmd` field: use the first alias (e.g., `rg` for ripgrep) or the tool key itself.
- Include `version` and `path` only when available (not null).
- Include `install` field only when tool is missing (install guidance from `getInstallCommand`).
- Always output as JSON via `output()` — no human-readable table mode for this command.
- Import `getInstallCommand` from `install-guidance.js` and `TOOLS` from `detector.js`.

**3. Export `cmdDetectTools`** from the module.

**4. Keep existing `cmdToolsStatus`** unchanged for backward compatibility of `util:tools`.
