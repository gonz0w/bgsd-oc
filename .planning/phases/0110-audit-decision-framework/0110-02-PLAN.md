**Candidates 3 & 4 from research — MEDIUM priority, ~30 min + ~20 min.**

**Candidate 3: progress.md route pre-computation**

The progress workflow (Routes A-F) evaluates deterministic conditions:
- Has UAT gaps → Route E
- summaries < plans → Route A (unexecuted plans exist)
- summaries = plans, plans > 0 → phase complete, then C or D based on phase position
- plans = 0 → Route B (needs planning)
- Between milestones → Route F

In the init handler for `bgsd-progress` (or the general enrichment path):
1. Count plans and summaries in the current phase directory using existing `parsePlans()` or equivalent
2. Check for UAT gap files
3. Compare current phase vs highest phase number
4. Check for ROADMAP.md existence (Route F detection)
5. Return `progress_route` as an object: `{ route: "A"|"B"|"C"|"D"|"E"|"F", next_plan_path?: string, next_phase?: number, summary: string }`

The `findPhaseInternal()` helper already provides most directory data. Use existing file-counting patterns from the enricher.

In `workflows/progress.md`: Add a section near the top (after the init block) that says:
```
## Pre-computed Route

Use `progress_route` from `<bgsd-context>` to determine which route to follow. The routing table below is kept as documentation.
```
Keep the full routing table (Routes A-F) as-is for documentation — it's the reference for what each route does. The LLM now skips the evaluation step and goes straight to the correct route section.

**Candidate 4: execute-plan.md pattern selection**

Pattern selection is deterministic: scan task types in the plan, classify checkpoints:
- No checkpoints → Pattern A (fully autonomous)
- Only `checkpoint:human-verify` → Pattern B (segment-by-segment)
- Has `checkpoint:decision` → Pattern C (standard flow)

In the init handler for `bgsd-execute-phase`:
1. Use `parseTasksFromPlan()` from `src/lib/orchestration.js` (or equivalent) to extract task types from the plan
2. Check for checkpoint task types
3. Return `execution_pattern` as: `{ pattern: "A"|"B"|"C", checkpoint_count: number, has_decision_checkpoints: boolean }`

In `workflows/execute-plan.md`: Add direction to use `execution_pattern` from `<bgsd-context>` for pattern selection. Keep the pattern descriptions as documentation.
