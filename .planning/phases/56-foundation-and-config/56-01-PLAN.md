<task type="auto">
  <name>Task 2: Create research.js with CLI and MCP detection functions</name>
  <files>src/commands/research.js</files>
  <action>
Create `src/commands/research.js` as a new command module following the pattern of `src/commands/env.js` and `src/commands/mcp.js`.

**Imports needed:**
- `fs`, `path` from Node.js
- `checkBinary` from `../commands/env` (the existing binary detection function)
- `loadConfig` from `../lib/config`
- `debugLog` from `../lib/output`

**Function 1: `detectCliTools(cwd)`**
Detects CLI research tools. Returns object:
```js
{
  'yt-dlp': { available: bool, version: string|null, path: string|null, install_hint: 'pip install yt-dlp' },
  'notebooklm-py': { available: bool, version: string|null, path: string|null, install_hint: 'pip install notebooklm-py' }
}
```

Implementation:
- Load config via `loadConfig(cwd)`
- For yt-dlp: if `config.ytdlp_path` is non-empty, check that path exists; otherwise call `checkBinary('yt-dlp', '--version')`
- For notebooklm-py: if `config.nlm_path` is non-empty, check that path; otherwise call `checkBinary('notebooklm-py', '--version')`
  - Note: notebooklm-py binary name may be `nlm` — try both: first `notebooklm-py`, then `nlm` as fallback
- Use the existing `checkBinary` function from env.js (already handles `which`, version parsing, 3s timeout)

**Function 2: `detectMcpServers(cwd)`**
Detects configured MCP research servers by reading the editor's MCP config file. Returns object:
```js
{
  'brave-search': { configured: bool, enabled: bool, name_match: string|null },
  'context7': { configured: bool, enabled: bool, name_match: string|null },
  'exa': { configured: bool, enabled: bool, name_match: string|null }
}
```

Implementation:
- Load config, get `mcp_config_path`
- If empty, try smart defaults: `~/.config/oc/config.json` (the host editor's main config — check for `mcpServers` key inside), then common patterns
- Read the MCP config JSON file, look for a `mcpServers` object (or `mcp.servers` — handle both shapes)
- For each server entry, do case-insensitive keyword matching on the server name:
  - Name contains "brave" → brave-search
  - Name contains "context7" → context7
  - Name contains "exa" → exa
- Check for `"disabled": true` in server config — report as configured but not enabled
- If file unreadable (missing, permissions): return all servers as `{ configured: false, enabled: false }` + set a `warning` field explaining why ("MCP config not found at [path]")
- Use `debugLog('research.mcp', ...)` for debug output

**Module exports:**
```js
module.exports = { detectCliTools, detectMcpServers };
```

Do NOT add command handler functions yet (that's Plan 02). This module provides only the detection infrastructure.
