**In `src/commands/research.js`:**

1. Add `status` to the require from `../lib/output` (currently imports `output, debugLog` — add `status`).

2. Add `collectWebSources(cwd, query, config, timeout)` function:
   - Uses `execFileSync(process.execPath, [process.argv[1], 'util:websearch', query, '--limit', '5'], { encoding: 'utf-8', timeout, stdio: 'pipe', cwd })` to call the existing websearch command as subprocess.
   - Parses JSON stdout. If `data.results` exists, maps each to `{ type: 'web', title, url, snippet, source: 'brave_search' }`.
   - Returns array of source objects (empty array on failure).
   - Wraps in try/catch — any failure returns `[]` silently (logged via debugLog).

3. Add `collectYouTubeSources(cwd, query, config, timeout)` function:
   - Step 1: Calls `execFileSync(process.execPath, [process.argv[1], 'research:yt-search', query, '--count', '3'], ...)` to search.
   - Step 2: If results found, calls `execFileSync(process.execPath, [process.argv[1], 'research:yt-transcript', topVideo.id], ...)` for top result ONLY (expensive — 1 video only).
   - Returns array with search result metadata + optional transcript for top video.
   - Wraps in try/catch — any failure returns `[]`.

4. Add `escapeXmlAttr(str)` function: replaces `&`, `"`, `<`, `>` with XML entities.

5. Add `truncateTranscript(text, maxChars)` function: truncates at word boundary, appends `\n[...transcript truncated]` if truncated. Default maxChars = 3000.

6. Add `formatSourcesForAgent(sources, query)` function:
   - If no sources, returns empty string `""`.
   - Builds XML-tagged string: `<collected_sources>`, `<research_query>`, then `<source type="..." title="..." url="...">` for each source. YouTube sources include `channel` attribute.
   - Web source content = snippet. YouTube source content = transcript (or `[Video: title — transcript unavailable]`).
   - Closes with `</collected_sources>`.

7. Add `formatCollect(data)` TTY formatter function:
   - Shows banner "Research Collection".
   - Shows tier number and name (color-coded: green for 1-2, yellow for 3, red for 4).
   - If skipped, shows skip reason.
   - Shows source table (Type, Title, Source columns) using formatTable.
   - Shows timing breakdown (web_ms, youtube_ms, total_ms) formatted as seconds.

8. Add `cmdResearchCollect(cwd, args, raw)` command handler:
   - Load config via `loadConfig(cwd)`. Check `ragEnabled = config.rag_enabled !== false`.
   - Parse `--quick` flag from args.
   - If `--quick` or `!ragEnabled`: output tier 4 result with `skipped: 'quick_flag'` or `'rag_disabled'`, empty sources, empty timing. Return early.
   - Detect tools: `detectCliTools(cwd)`, `detectMcpServers(cwd)`, `calculateTier(cliTools, mcpServers, ragEnabled)`.
   - Extract query from positional args (filter out `--` prefixed args). If empty query, output error and return.
   - Allocate per-stage timeout: `stageTimeout = Math.max(5000, Math.floor((config.rag_timeout || 30) * 1000 / 2))` — split total budget between stages.
   - Stage 1: `status('[1/3] Collecting web sources...')`, call `collectWebSources`, record `timing.web_ms`.
   - Stage 2: If `cliTools['yt-dlp'].available`, `status('[2/3] Searching YouTube...')`, call `collectYouTubeSources`, record `timing.youtube_ms`. Else `status('[2/3] YouTube: skipped (yt-dlp not installed)')`, set `timing.youtube_ms = 0`.
   - Stage 3: `status('[3/3] Context7: available to agent directly via MCP')`, set `timing.context7_available = mcpServers['context7']?.configured && mcpServers['context7']?.enabled`.
   - Compute `timing.total_ms`.
   - Build `agent_context = formatSourcesForAgent(sources, query)`.
   - `status('Research collection complete: N sources, Tier X (Yms)')`.
   - Output result object via `output(result, { formatter: formatCollect, raw })`.

9. Update `module.exports` to include: `cmdResearchCollect` (add alongside existing exports).
