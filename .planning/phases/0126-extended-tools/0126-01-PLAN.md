**env.js — yq for docker-compose service extraction (detectInfraServices, ~line 537):**

1. Add `const { parseYAML } = require('../lib/cli-tools');` at the top (alongside existing `searchRipgrep` and `transformJson` imports from cli-tools — extend the existing destructuring).

2. In `detectInfraServices()`, after confirming the compose file exists, replace the current two-branch approach (ripgrep check + regex parsing in both branches) with a simpler yq-first approach:
   - First try `parseYAML(content)` on the file content. If `parsed.success` and `parsed.result` has a `.services` key, extract service names with `Object.keys(parsed.result.services)`.
   - If parseYAML fails or returns no services, fall back to the existing regex approach (the `servicesMatch` regex pattern) for backward compat.
   - Keep the existing ripgrep optimization as an early check: if ripgrep finds no `image:|build:` lines, skip the file entirely (no services). This pre-filter is still valuable even with yq.
   - The flow becomes: ripgrep pre-filter → yq structured parse → regex fallback.

   IMPORTANT: `parseYAML` uses `withToolFallback` internally — when yq unavailable, its JS fallback handles basic YAML. But the basic fallback may not handle complex docker-compose structure (nested services with indentation). So keep the regex as an ultimate fallback after the parseYAML path.

3. In `detectMonorepo()` for pnpm-workspace.yaml parsing (~line 649-657):
   - After reading the file content, try `parseYAML(content)` first.
   - If `parsed.success` and `parsed.result.packages`, extract members directly: `const members = parsed.result.packages;`
   - If parseYAML fails, fall back to the existing regex approach.
   - This replaces the brittle regex-based `packages:` block extraction with structured parsing.

Per CONTEXT.md locked decision: **Silent fallback** — no user-facing message when yq unavailable. The `usedFallback` and `guidance` fields in the parseYAML result must NOT be printed or logged to the user. The workflow just works with whichever backend ran.

**features.js — bat for session diff display (cmdSessionDiff, ~line 19-79):**

4. Add `const { catWithHighlight, isBatAvailable } = require('../lib/cli-tools');` at the top alongside existing imports.

5. In `cmdSessionDiff()`, after building the `changes` array from git log, add bat-enhanced diff display:
   - After the existing `filesChanged` extraction (around line 56-58), if bat is available (`isBatAvailable()`), generate a unified diff for display:
     - Use `execGit(cwd, ['diff', '--stat', `--since=${since}`, '--', '.planning/'])` to get a diff stat summary.
     - If the stat output is non-empty, write it to a temp file with `.diff` extension, then call `catWithHighlight(tempFile, { language: 'diff', style: 'numbers,grid', color: 'auto' })`.
     - Add the highlighted result as a `diff_highlighted` field in the output JSON.
     - Clean up the temp file in a finally block.
   - If bat is unavailable, simply omit the `diff_highlighted` field (silent fallback — no message).
   - The existing JSON output structure remains unchanged — `diff_highlighted` is additive.

   IMPORTANT: Use `isToolEnabled('bat')` from fallback.js (not `isBatAvailable()`) to respect config toggles. Only proceed with bat display if the tool is both enabled AND available.

**misc.js — bat for rollback info diff display (cmdRollbackInfo, ~line 1608-1616):**

6. Add `const { catWithHighlight } = require('../lib/cli-tools');` and `const { isToolEnabled } = require('../lib/cli-tools/fallback');` at the top alongside existing imports.

7. In `cmdRollbackInfo()`, after the `diffSummary()` call (~line 1614-1616), add bat-enhanced diff:
   - If `isToolEnabled('bat')` and diff has file changes, create a formatted diff summary string from the diff data (listing changed files with insertions/deletions).
   - Write to a temp `.diff` file, highlight with `catWithHighlight(tempFile, { language: 'diff', style: 'numbers,grid' })`, and include as `diff_highlighted` in the output.
   - If bat unavailable or disabled, omit `diff_highlighted` (silent fallback).
   - Clean up temp file in a finally block.

Per CONTEXT.md: bat is for **display output only**. Internal file reads and data extraction stay as readFileSync. bat adds visual enhancement, not functional changes.
