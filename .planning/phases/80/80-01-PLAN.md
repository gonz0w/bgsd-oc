### Tasks

**<task>**
<files>src/lib/cache.js</files>
<action>

1. Add env var guard at top of SQLiteBackend constructor:
   ```javascript
   this.useStatementCache = process.env.BGSD_SQLITE_STATEMENT_CACHE !== '0';
   ```

2. In `_initSchema()`, after creating tables, set up statement cache if enabled:
   ```javascript
   if (this.useStatementCache) {
     try {
       this._stmtStore = this.db.createTagStore();
       this._cachedGet = this._stmtStore.compile('SELECT * FROM file_cache WHERE key = ?');
       this._cachedSet = this._stmtStore.compile('INSERT OR REPLACE INTO file_cache (key, value, mtime, created, accessed) VALUES (?, ?, ?, ?, ?)');
       this._cachedUpdate = this._stmtStore.compile('UPDATE file_cache SET accessed = ? WHERE key = ?');
     } catch (e) {
       this.useStatementCache = false;
     }
   }
   ```

3. Modify `get()` to use cached statement when available:
   ```javascript
   get(key) {
     try {
       const stmt = this.useStatementCache ? this._cachedGet : this.db.prepare('SELECT * FROM file_cache WHERE key = ?');
       const row = stmt.get(key);
       // ... rest of get logic
     }
   }
   ```

4. Modify `set()` and other methods similarly

5. Add debug output when BGSD_DEBUG=1 showing statement cache status
