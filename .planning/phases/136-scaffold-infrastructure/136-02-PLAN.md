# Plan 02: plan:generate Command

## Objective

Implement the `plan:generate --phase <N>` CLI command that reads the phase definition from ROADMAP.md and produces a pre-filled PLAN.md scaffold. Data sections (frontmatter, objective, requirement links, file paths from previous summaries) are pre-filled by the CLI; judgment sections (task breakdown, verification strategy, success criteria details) are marked for LLM completion. Re-running on an existing file preserves LLM-written judgment sections.

## Tasks

### Task 1: Implement plan:generate command and wire routing

- **Type:** auto
- **Files:**
  - `src/commands/scaffold.js` (create)
  - `src/router.js` (modify)

- **Action:**

  1. **Create `src/commands/scaffold.js`** with `cmdPlanGenerate(cwd, args, raw)`:

     **Argument parsing:**
     - `--phase <N>` (required) — phase number
     - `--plan <NN>` (optional, default: auto-detect next plan number from existing files)
     - `--type <type>` (optional, default: `standard`)
     - `--wave <N>` (optional, default: `1`)

     **Data extraction (pre-fill from deterministic sources):**
     - Call `findPhaseInternal(cwd, phaseNum)` to resolve phase directory
     - Call `cmdRoadmapGetPhase(cwd, phaseNum, true)` or parse ROADMAP.md directly to extract: `goal`, `requirements` (comma-separated IDs), `success_criteria` (numbered list), `depends_on`
     - Scan phase directory for existing PLAN.md files to auto-detect next plan number (e.g., if `136-01-PLAN.md` and `136-02-PLAN.md` exist, next is `03`)
     - If previous plan summaries exist in the phase, extract `provides` and `files_modified` for dependency context

     **Scaffold generation with markers:**
     - **Frontmatter** (all `<!-- data -->`): `phase`, `plan`, `title` (TODO placeholder), `type`, `requirements` (from roadmap), `depends_on` (from roadmap + previous plan), `files_modified` (empty array), `estimated_tasks`, `wave`
     - **Objective section** (`<!-- data -->`): Pre-fill with phase goal from ROADMAP.md. Add "This plan contributes to:" + goal text.
     - **Context section** (`<!-- data -->`): Auto-generate `@` file references to ROADMAP.md, STATE.md, and any previous plan summaries in this phase
     - **Requirements section** (`<!-- data -->`): List requirement IDs from roadmap with their text from REQUIREMENTS.md if available
     - **Must-Haves section** (`<!-- judgment -->`): Empty template with `truths: []`, `artifacts: []`, `key_links: []` for LLM to fill
     - **Tasks section** (`<!-- judgment -->`): Task template structure with `TODO:` markers for LLM to fill
     - **Verification section** (`<!-- judgment -->`): TODO marker for LLM to describe verification approach
     - **Success Criteria section** (`<!-- data -->` + `<!-- judgment -->`): Pre-fill with success criteria from ROADMAP.md, with TODO markers for plan-specific refinement

     **Idempotent merge:**
     - Before writing, check if the target file exists
     - If it exists, read it and call `mergeScaffold(existing, fresh, PLAN_JUDGMENT_SECTIONS)` from `src/lib/scaffold.js`
     - `PLAN_JUDGMENT_SECTIONS` = `['Must-Haves', 'Tasks', 'Verification']`
     - Write the merged result (preserving LLM judgment content, refreshing data)
     - If file doesn't exist, write fresh scaffold directly

     **Output:** JSON with `{ created: true, path, phase, plan, sections: { data: N, judgment: N } }`

  2. **Wire routing in `src/router.js`:**
     - Add `lazyScaffold()` function in the lazy-loader section (around line 84-114) pointing to `require('./commands/scaffold')`
     - In the `case 'plan':` block (line 381), add `else if (subcommand === 'generate')` branch that parses `--phase`, `--plan`, `--type`, `--wave` from `restArgs` and calls `lazyScaffold().cmdPlanGenerate(cwd, options, raw)`

  3. **Export** `cmdPlanGenerate` and the `PLAN_JUDGMENT_SECTIONS` constant from `scaffold.js`.

- **Verify:** `node bin/bgsd-tools.cjs plan:generate --phase 136 --raw` produces JSON output with `created: true`. The generated PLAN.md file exists in the phase directory with `<!-- data -->` and `<!-- judgment -->` markers. Frontmatter contains phase goal-derived objective.

- **Done:** `plan:generate --phase <N>` command works end-to-end. Produces pre-filled PLAN.md with data/judgment separation. Routed through `plan:generate` in the CLI. Handles auto-numbering, idempotent merge, and roadmap data extraction.
