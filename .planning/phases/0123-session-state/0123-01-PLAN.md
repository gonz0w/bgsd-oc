<task type="auto">
  <name>Task 1: Add MIGRATIONS[4] for session state tables and sync db-cache.js</name>
  <files>src/lib/db.js, src/plugin/lib/db-cache.js</files>
  <action>
Add MIGRATIONS[4] (migration_v5) to src/lib/db.js creating six session state tables:

1. **session_state** — current position (one row per cwd):
   ```sql
   CREATE TABLE IF NOT EXISTS session_state (
     cwd            TEXT PRIMARY KEY,
     phase_number   TEXT,
     phase_name     TEXT,
     total_phases   INTEGER,
     current_plan   TEXT,
     status         TEXT,
     last_activity  TEXT,
     progress       INTEGER,
     milestone      TEXT,
     data_json      TEXT
   );
   ```

2. **session_metrics** — performance metrics (append-only):
   ```sql
   CREATE TABLE IF NOT EXISTS session_metrics (
     id          INTEGER PRIMARY KEY AUTOINCREMENT,
     cwd         TEXT NOT NULL,
     milestone   TEXT,
     phase       TEXT,
     plan        TEXT,
     duration    TEXT,
     tasks       INTEGER,
     files       INTEGER,
     test_count  INTEGER,
     timestamp   TEXT,
     data_json   TEXT
   );
   CREATE INDEX IF NOT EXISTS idx_session_metrics_cwd ON session_metrics(cwd);
   CREATE INDEX IF NOT EXISTS idx_session_metrics_phase ON session_metrics(phase);
   ```

3. **session_decisions** — individual decisions (append-only, one per row):
   ```sql
   CREATE TABLE IF NOT EXISTS session_decisions (
     id          INTEGER PRIMARY KEY AUTOINCREMENT,
     cwd         TEXT NOT NULL,
     milestone   TEXT,
     phase       TEXT,
     summary     TEXT,
     rationale   TEXT,
     timestamp   TEXT,
     data_json   TEXT
   );
   CREATE INDEX IF NOT EXISTS idx_session_decisions_cwd ON session_decisions(cwd);
   CREATE INDEX IF NOT EXISTS idx_session_decisions_phase ON session_decisions(phase);
   ```

4. **session_todos** — todos with extended metadata:
   ```sql
   CREATE TABLE IF NOT EXISTS session_todos (
     id          INTEGER PRIMARY KEY AUTOINCREMENT,
     cwd         TEXT NOT NULL,
     text        TEXT NOT NULL,
     priority    TEXT,
     category    TEXT,
     status      TEXT NOT NULL DEFAULT 'pending',
     created_at  TEXT,
     completed_at TEXT,
     data_json   TEXT
   );
   CREATE INDEX IF NOT EXISTS idx_session_todos_cwd ON session_todos(cwd);
   CREATE INDEX IF NOT EXISTS idx_session_todos_status ON session_todos(status);
   ```

5. **session_blockers** — blockers with lifecycle tracking:
   ```sql
   CREATE TABLE IF NOT EXISTS session_blockers (
     id             INTEGER PRIMARY KEY AUTOINCREMENT,
     cwd            TEXT NOT NULL,
     text           TEXT NOT NULL,
     status         TEXT NOT NULL DEFAULT 'open',
     created_at     TEXT,
     resolved_at    TEXT,
     resolution     TEXT,
     linked_decision_id INTEGER,
     data_json      TEXT
   );
   CREATE INDEX IF NOT EXISTS idx_session_blockers_cwd ON session_blockers(cwd);
   CREATE INDEX IF NOT EXISTS idx_session_blockers_status ON session_blockers(status);
   ```

6. **session_continuity** — session continuity (one row per cwd):
   ```sql
   CREATE TABLE IF NOT EXISTS session_continuity (
     cwd          TEXT PRIMARY KEY,
     last_session TEXT,
     stopped_at   TEXT,
     next_step    TEXT,
     data_json    TEXT
   );
   ```

Then update src/plugin/lib/db-cache.js:
- Rename SCHEMA_V4_SQL to SCHEMA_V5_SQL
- Append all six session tables and indexes to the SQL string
- Update _ensureSchema version check from `>= 4` to `>= 5`
- Update PRAGMA user_version from 4 to 5

<task type="auto">
  <name>Task 2: Add PlanningCache session state methods (CJS + ESM)</name>
  <files>src/lib/planning-cache.js, src/plugin/lib/db-cache.js</files>
  <action>
Add session state methods to PlanningCache in both src/lib/planning-cache.js (CJS) and src/plugin/lib/db-cache.js (ESM). All methods follow the established pattern: _isMap() guard returning null, lazy prepared statements via _stmt(), try/catch wrapping.

**Position methods:**
- `storeSessionState(cwd, state)` — Upsert into session_state. `state` is an object with fields: phase_number, phase_name, total_phases, current_plan, status, last_activity, progress, milestone. Store full object in data_json for round-tripping.
- `getSessionState(cwd)` — Get session_state row for cwd. Return null on miss.
- `migrateStateFromMarkdown(cwd, parsed)` — One-time import from parsed STATE.md data. Takes the parsed state object (from parseState or stateExtractField equivalent) and populates session_state, session_decisions, session_metrics, session_todos, session_blockers, session_continuity. Idempotent — skip if session_state row already exists for cwd. Parse decisions from `### Decisions` section, metrics from `## Performance Metrics`, todos from `### Pending Todos`, blockers from `### Blockers/Concerns`, continuity from `## Session Continuity`.

**Metrics methods:**
- `writeSessionMetric(cwd, metric)` — Insert into session_metrics. `metric` has: milestone, phase, plan, duration, tasks, files, test_count, timestamp.
- `getSessionMetrics(cwd, options)` — Query session_metrics for cwd with optional phase filter and limit. Return { entries, total } or null.

**Accumulated context methods:**
- `writeSessionDecision(cwd, decision)` — Insert into session_decisions. `decision` has: milestone, phase, summary, rationale, timestamp.
- `getSessionDecisions(cwd, options)` — Query with optional phase filter, limit, offset. Return { entries, total } or null.
- `writeSessionTodo(cwd, todo)` — Insert into session_todos. `todo` has: text, priority, category, status, created_at.
- `getSessionTodos(cwd, options)` — Query with optional status filter ('pending'/'completed'). Return { entries, total } or null.
- `completeSessionTodo(cwd, id)` — Update session_todos SET status='completed', completed_at=now WHERE id=? AND cwd=?.
- `writeSessionBlocker(cwd, blocker)` — Insert into session_blockers. `blocker` has: text, status, created_at.
- `getSessionBlockers(cwd, options)` — Query with optional status filter ('open'/'resolved'). Return { entries, total } or null.
- `resolveSessionBlocker(cwd, id, resolution)` — Update session_blockers SET status='resolved', resolved_at=now, resolution=? WHERE id=? AND cwd=?.

**Session continuity methods:**
- `recordSessionContinuity(cwd, continuity)` — Upsert into session_continuity. `continuity` has: last_session, stopped_at, next_step.
- `getSessionContinuity(cwd)` — Get session_continuity row for cwd. Return null on miss.

All methods must have identical signatures in both CJS and ESM versions. ESM version uses the same _stmt() / _isMap() pattern already established in db-cache.js.
  </action>
