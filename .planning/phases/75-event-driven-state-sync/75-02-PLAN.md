**Create `src/plugin/idle-validator.js`** — debounced idle validation with auto-fix.

Factory function `createIdleValidator(cwd, notifier, fileWatcher, config)` returns:
- `onIdle()` — called on `session.idle` event. Implementation:
  1. Check `config.idle_validation.enabled` — bail if false
  2. Check `existsSync(join(cwd, '.planning'))` — bail silently if no project (per CONTEXT.md)
  3. Check cooldown: `Date.now() - lastValidation < cooldownMs` — bail if within cooldown
  4. Check `lastAutoFix > lastValidation` — bail if we auto-fixed and haven't seen user interaction yet (prevents fix loops)
  5. Check `validating` flag — bail if already running (re-entry guard)
  6. Check active execution: read STATE.md status field — if "executing" or similar, bail (per CONTEXT.md: skip during active plan execution)
  7. Set `validating = true`, run validation:
     a. **STATE.md + ROADMAP.md cross-check**: Parse state, get phase number, verify it exists in roadmap. Check status consistency.
     b. **Config.json schema check**: Verify parseConfig doesn't return pure defaults (which would mean corrupt file). Auto-fix by writing defaults if needed.
     c. **Stale progress detection**: If state.status contains "executing" or "in progress" but last git activity is older than `staleness_threshold_hours`, auto-fix status to "paused". Use `execSync('git log -1 --format=%ct')` for last git timestamp.
     d. **Phase completion detection**: Compare state before and after — if a phase just completed (was incomplete, now complete), fire EVNT-03 notification: `notifier.notify({ type: 'phase-complete', severity: 'warning', message: 'Phase N complete! Plans: X, Duration: Ymin', action: 'Next: /bgsd-plan-phase N+1' })`.
     e. **Progress bar fix**: If progress bar visual doesn't match percentage, auto-fix (pattern from bgsd-validate.js).
  8. If any auto-fix applied: `fileWatcher.trackSelfWrite(path)` before writing, write file, set `lastAutoFix = Date.now()`, fire minimal "State synced" notification via notifier (info severity).
  9. Set `validating = false`, update `lastValidation = Date.now()`.
- `onUserInput()` — resets the "waiting for user interaction after auto-fix" flag so validation resumes.

All operations wrapped in try/catch — never throws. Log errors via createLogger.

**Create `src/plugin/stuck-detector.js`** — tool call repetition tracker.

Factory function `createStuckDetector(notifier, config)` returns:
- `trackToolCall(input)` — called on `tool.execute.after`. Implementation:
  1. Extract `toolName = input.tool`, `error = input.error || null`, `args = JSON.stringify(input.args || {})`
  2. **Error loop detection** (EVNT-04): If error exists, build key = `${toolName}:${error}`. Increment counter in Map. If count >= `config.stuck_detection.error_threshold` (default 3):
     - Fire critical notification: `{ type: 'stuck-error', severity: 'critical', message: 'Stuck: ${toolName} failed ${count}x — "${error}"', action: 'Consider pausing for user input' }`
     - Inject system message to agent context (via notifier pending context queue)
     - Track escalation level — if this is 2nd stuck detection in session, include "ESCALATED" in message
  3. **Spinning detection**: Track last N tool calls as `[toolName, argsHash]` tuples. If same sequence of length >= `config.stuck_detection.spinning_threshold` (default 5) repeats consecutively, fire warning notification: `{ type: 'stuck-spinning', severity: 'warning', message: 'Spinning: same ${len}-call sequence repeated ${repeatCount}x' }`
  4. **Reset on success**: If no error, clear error streak Map. (Don't clear spinning history — spinning can occur with successful calls.)
- `onUserInput()` — resets all counters (error streak, escalation level, spinning history). Called when user types.

Import: createLogger from logger.js for error logging.
