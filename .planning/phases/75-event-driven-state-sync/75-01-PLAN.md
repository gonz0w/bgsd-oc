  <name>Task 1: Create notification delivery system</name>
  <files>src/plugin/notification.js</files>
  <action>
Create `src/plugin/notification.js` — dual-channel notification module.

Factory function `createNotifier($, directory)` returns an object with:
- `notify(notification)` — routes by severity: critical/warning → OS notification via `$` shell API (osascript on macOS, notify-send on Linux); all severities → queue for context injection. Stores in ring buffer history (last 20). Respects DND mode (suppress routine, pass critical through). Applies rate limiting (configurable, default 5/minute — drop excess). Deduplicates related events within a time window (match by type+message within debounce window).
- `drainPendingContext()` — returns and clears pending context-injection notifications. Returns array of `{type, severity, message, timestamp}` objects.
- `getHistory()` — returns last 20 notifications (read-only copy).
- `setDnd(enabled)` — toggle DND mode. When DND ends, return summary count of suppressed notifications.
- `resetCounters()` — reset rate limit counters (called on user input).

Notification shape: `{ type: string, severity: 'info'|'warning'|'critical', message: string, action?: string }`.

OS notification delivery (per RESEARCH.md patterns):
- macOS: `await $\`osascript -e 'display notification "${message}" with title "bGSD"'\`.quiet()`
- Linux: `await $\`notify-send "bGSD" "${message}"\`.quiet()`
- Both wrapped in try/catch — failure is silent (logged only).
- Detection: `process.platform === 'darwin'` for macOS, else Linux.

Context injection format (per CONTEXT.md decision):
`<bgsd-notification type="${type}" severity="${severity}">${message}</bgsd-notification>`

DND mode (per CONTEXT.md):
- Suppress routine (info) notifications when DND active
- Critical (stuck detection) always passes through
- When DND ends with `setDnd(false)`, show one summary notification pointing to `/bgsd-notifications`

Rate limiting: sliding window per minute. Track timestamps, drop oldest when window exceeds cap.

Import only: `createLogger` from logger.js (for error logging on OS notification failure).
No imports from safe-hook.js (wrapping handled at registration in index.js).

<task type="auto">
  <name>Task 2: Create file watcher with cache invalidation</name>
  <files>src/plugin/file-watcher.js</files>
  <action>
Create `src/plugin/file-watcher.js` — file system watcher with self-change tracking.

Factory function `createFileWatcher(cwd, options)` returns an object with:
- `start()` — begins watching `.planning/` directory. Uses `fs.watch` with `{ recursive: true }` and AbortController for clean shutdown. On file change events:
  1. Check if path is in `selfWrites` set → skip if self-change
  2. Apply debounce (configurable, default 200ms) — coalesce rapid events
  3. Call `invalidateAll(cwd)` from parsers/index.js
  4. Also watch for `config.json` changes in `.planning/` (already covered by recursive watch)
- `stop()` — aborts the AbortController, cleans up watcher
- `trackSelfWrite(filePath)` — adds path to self-writes set, auto-clears after 200ms
- `isWatching()` — returns current watcher state (boolean)

Per CONTEXT.md decisions:
- Watch for creates, modifications, and deletes (all fs.watch events)
- Start watching after `.planning/` directory confirmed to exist — if no `.planning/`, log and return without watching
- Reasonable cap on watched paths (~500) — log warning if too many events suggest excessive paths
- On watcher errors (EPERM, etc.): log and degrade gracefully — don't crash
- External edits trigger cache invalidation only — no validation, no immediate re-read
- Self-change tracking: Set with 200ms auto-clear timeout (per RESEARCH.md Pattern 2)

Debounce implementation: accumulate events for `debounceMs`, then call invalidateAll once with accumulated paths. Use setTimeout/clearTimeout pattern.

Import: `invalidateAll` from `./parsers/index.js`, `createLogger` from `./logger.js`, `existsSync` from 'fs'.
  </action>
  <verify>
Run: `node -e "import('./src/plugin/file-watcher.js').then(m => { const w = m.createFileWatcher('/tmp/nonexistent'); w.start(); console.log('watching:', w.isWatching()); w.stop(); })"`
Should print 'watching: false' (since /tmp/nonexistent/.planning doesn't exist).
  </verify>
  <done>
createFileWatcher() factory returns start/stop/trackSelfWrite/isWatching. Watches .planning/ recursively. Debounces events. Calls invalidateAll on external changes. Ignores self-writes. Degrades gracefully on errors. Respects path cap.
