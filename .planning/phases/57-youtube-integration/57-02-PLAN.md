  <name>Task 1: Implement yt-transcript command with VTT parsing</name>
  <files>src/commands/research.js, src/router.js, src/lib/constants.js</files>
  <action>
Add `cmdResearchYtTranscript(cwd, args, raw)` to `src/commands/research.js`. This function:

1. **Check yt-dlp availability first** using `detectCliTools(cwd)`. If unavailable, return:
   `{ error: 'yt-dlp not installed', install_hint: 'pip install yt-dlp', available: false }` — graceful JSON, no process exit.

2. **Parse args:**
   - Positional arg: video ID (required — error if missing). Accept both bare ID (`dQw4w9WgXcQ`) and full URL (`https://www.youtube.com/watch?v=dQw4w9WgXcQ`) — extract ID from URL if needed.
   - `--timestamps` flag: if present, preserve timestamps in output (default: strip)
   - `--lang LANG` (default: `en` — language code for subtitle preference)

3. **Download subtitles to temp dir** using `execFileSync`:
   ```
   const tmpDir = fs.mkdtempSync(path.join(os.tmpdir(), 'gsd-yt-'));
   execFileSync(ytdlpPath, [
     '--write-sub',
     '--write-auto-sub',
     '--sub-lang', lang + ',en,en-auto',
     '--sub-format', 'vtt',
     '--skip-download',
     '--no-warnings',
     '-o', path.join(tmpDir, '%(id)s'),
     videoUrl
   ], { encoding: 'utf-8', timeout: 30000, stdio: 'pipe' })
   ```
   Where `ytdlpPath` is from config override or 'yt-dlp', and `videoUrl` is `https://www.youtube.com/watch?v=${videoId}`.

4. **Find the VTT file** in tmpDir. yt-dlp writes files like `{id}.{lang}.vtt`. List dir, find `.vtt` files, prefer exact lang match over auto-generated.

5. **Parse VTT content** — implement `parseVtt(vttContent, keepTimestamps)`:
   - Skip the `WEBVTT` header line and any metadata lines
   - For each cue block: extract text content (lines after timestamp line)
   - Strip HTML tags (`<c>`, `</c>`, `<b>`, etc.) from text
   - Strip VTT positioning tags (anything in `<>` angle brackets)
   - Deduplicate consecutive identical lines (VTT auto-subs repeat lines across overlapping cues)
   - If `keepTimestamps` is false (default): output only text, joined with spaces, paragraphs separated by newlines
   - If `keepTimestamps` is true: output `[HH:MM:SS] text` format for each cue
   - Clean up excessive whitespace (multiple spaces → single, multiple newlines → double)

6. **Handle no-subtitles case:** If no VTT file found in tmpDir, return:
   `{ video_id, error: null, transcript: null, message: 'No subtitles available for this video', has_subtitles: false }`
   Per CONTEXT.md: structured response, no fake fallbacks.

7. **Clean up tmpDir** after reading (rm -rf equivalent using fs.rmSync with recursive).

8. **Return structured result:**
   ```json
   {
     "video_id": "...",
     "has_subtitles": true,
     "language": "en",
     "auto_generated": true|false,
     "transcript": "full clean text...",
     "word_count": N,
     "char_count": N
   }
   ```
   Always return the full transcript per CONTEXT.md — no truncation.

9. **Add formatter** `formatYtTranscript(data)` for TTY output: show metadata header (video_id, language, word count), then transcript text (first 2000 chars with "... truncated for display" note if longer — display only, JSON always has full text).

10. **Wire into router.js:** In BOTH research namespace handlers, add `yt-transcript` routing to `lazyResearch().cmdResearchYtTranscript()`. Update error messages and available subcommand lists.

11. **Add COMMAND_HELP** entries in `src/lib/constants.js` for `research:yt-transcript` and `research yt-transcript`.

Export `cmdResearchYtTranscript` and `parseVtt` from research.js module.exports. Export `parseVtt` to enable unit testing of VTT parsing independently.
  </action>
