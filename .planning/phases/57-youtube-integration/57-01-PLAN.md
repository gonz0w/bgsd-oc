  <name>Task 1: Implement yt-search command with quality scoring and filtering</name>
  <files>src/commands/research.js, src/router.js, src/lib/constants.js</files>
  <action>
Add `cmdResearchYtSearch(cwd, args, raw)` to `src/commands/research.js`. This function:

1. **Check yt-dlp availability first** using `detectCliTools(cwd)`. If `yt-dlp` is not available, return structured error:
   `{ error: 'yt-dlp not installed', install_hint: 'pip install yt-dlp', available: false }` — do NOT call `error()` (which exits), return graceful JSON.

2. **Parse args:**
   - Positional arg: search query (required — error if missing)
   - `--count N` (default: 10, pre-filter count)
   - `--max-age DAYS` (default: 730 = ~2 years)
   - `--min-duration SEC` (default: 300 = 5 min)
   - `--max-duration SEC` (default: 3600 = 60 min)
   - `--min-views N` (default: 0 = no minimum)

3. **Execute yt-dlp search** using `execFileSync` (NOT `execSync` — follow `git.js` pattern):
   ```
   execFileSync('yt-dlp', [
     `ytsearch${count}:${query}`,
     '--dump-json',
     '--flat-playlist',
     '--no-download',
     '--no-warnings'
   ], { encoding: 'utf-8', timeout: 30000, stdio: 'pipe' })
   ```
   Each line of output is a separate JSON object (NDJSON). Parse each line with `JSON.parse()`.
   Wrap in try/catch — on failure, return `{ error: 'yt-dlp search failed', details: err.message }`.

4. **Extract structured fields** from each yt-dlp JSON result:
   - `id` (video ID)
   - `title`
   - `channel` (from `uploader` or `channel`)
   - `duration` (seconds, null if missing)
   - `view_count` (null if missing — never omit the field per CONTEXT.md)
   - `upload_date` (from `upload_date` field, format YYYYMMDD → ISO date string)
   - `url` (construct `https://www.youtube.com/watch?v=${id}`)
   - `description` (first 200 chars, null if missing)

5. **Apply filters** (post-extraction, pre-scoring):
   - Recency: if `upload_date` exists and is older than `--max-age` days from now, exclude
   - Duration: if `duration` exists and is outside `[min-duration, max-duration]` range, exclude
   - Views: if `view_count` exists and is below `--min-views`, exclude
   - If a field is null, do NOT filter on that dimension (keep the result)

6. **Compute quality score** (0-100) for each surviving result. Algorithm (agent's discretion per CONTEXT.md):
   - Recency component (0-40): Linear decay from 40 (today) to 0 (max-age days ago). Null upload_date → 20 (neutral).
   - View component (0-30): Log-scale. `min(30, log10(view_count + 1) * 6)`. Null → 10 (neutral).
   - Duration component (0-30): Bell curve centered at 15-20 min. Peak = 30 for 15-20 min range, drops linearly to 10 at bounds (5 min, 60 min). Null → 15 (neutral).
   - Round to integer. Clamp to [0, 100].
   - This actively demotes likely-clickbait (high views + very short) per CONTEXT.md.

7. **Sort by quality score** descending. Return:
   ```json
   {
     "query": "original query",
     "pre_filter_count": N,
     "post_filter_count": M,
     "results": [{ id, title, channel, duration, view_count, upload_date, url, description, quality_score }]
   }
   ```

8. **Add formatter** `formatYtSearch(data)` for TTY output using existing format.js primitives (banner, sectionHeader, formatTable). Show results as a table with score, title (truncated to 50 chars), channel, duration (formatted as M:SS), views (formatted with K/M suffixes), and date.

9. **Wire into router.js:** In BOTH research namespace handlers (colon-namespaced at ~line 700 and legacy at ~line 1566), add `yt-search` as a subcommand routing to `lazyResearch().cmdResearchYtSearch(cwd, restArgs/args.slice(2), raw)`. Update the error message to include `yt-search` in available subcommands.

10. **Add COMMAND_HELP** entry in `src/lib/constants.js` for `research:yt-search` and `research yt-search`.

Export `cmdResearchYtSearch` from research.js module.exports.
