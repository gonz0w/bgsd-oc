# Analyze conditionals for impossible branches
1. Create script to find:
   - if (false) { code } - obviously dead
   - if (true) { code } - but other branch is dead
   - Logical contradictions: if (x && !x)
   - Always-true comparisons: if (x === x), if (x != x)
   - Null checks on constants: if (null && x)
2. Also analyze switch statements:
   - Cases with constant values
   - Default after return
3. Output to dead-branches.json

Use AST to find:
- IfStatement with Literal false test
- IfStatement where consequent has return/throw and alternate is also dead
- BinaryExpression with always-true/false evaluation
