Extend src/lib/cache.js with research caching. Do NOT change existing file_cache table or methods.

Add module-level: const researchStats = { hits: 0, misses: 0 };

In SQLiteBackend._initSchema(), add after the file_cache CREATE TABLE:
  CREATE TABLE IF NOT EXISTS research_cache (
    key TEXT PRIMARY KEY,
    value TEXT NOT NULL,
    created REAL NOT NULL,
    accessed REAL NOT NULL,
    expires REAL NOT NULL
  )

Add to SQLiteBackend:

getResearch(key): SELECT from research_cache WHERE key=?. If not found: researchStats.misses++, return null. If Date.now() > row.expires: DELETE the row, researchStats.misses++, return null. UPDATE accessed=Date.now(). researchStats.hits++, return JSON.parse(row.value). Wrap in try/catch returning null on error.

setResearch(key, value, ttlMs=3600000): Serialize value=JSON.stringify(value). LRU eviction: if COUNT(*) from research_cache >= this.maxSize, delete oldest by accessed ASC LIMIT 1. INSERT OR REPLACE (key, value, created=now, accessed=now, expires=now+ttlMs). Silently fail on error.

clearResearch(): DELETE FROM research_cache. Silently fail on error.

statusResearch(): SELECT COUNT(*) from research_cache. Return { count, hits: researchStats.hits, misses: researchStats.misses }. On error return { count: 0, hits, misses }.

Add to MapBackend (use a separate this.researchMap = new Map() initialized in constructor):

getResearch(key): Check this.researchMap.get(key). If not found: researchStats.misses++, return null. If Date.now() > entry.expires: this.researchMap.delete(key), researchStats.misses++, return null. LRU re-insert (delete+set). researchStats.hits++, return JSON.parse(entry.value).

setResearch(key, value, ttlMs=3600000): LRU eviction at maxSize. Store { value: JSON.stringify(value), created: now, accessed: now, expires: now+ttlMs } in this.researchMap.

clearResearch(): this.researchMap.clear()

statusResearch(): return { count: this.researchMap.size, hits: researchStats.hits, misses: researchStats.misses }

Add to CacheEngine (pass-through to this.backend):
  getResearch(key) { return this.backend.getResearch(key); }
  setResearch(key, value, ttlMs) { this.backend.setResearch(key, value, ttlMs); }
  clearResearch() { this.backend.clearResearch(); }
  statusResearch() { return this.backend.statusResearch(); }

**A. src/commands/research.js — cache integration in cmdResearchCollect()**

At top of file, add lazy cache singleton (after existing requires):
  let _researchCache = null;
  function getResearchCache() {
    if (!_researchCache) {
      try { const {CacheEngine}=require('../lib/cache'); _researchCache=new CacheEngine(); }
      catch(e) { return null; }
    }
    return _researchCache;
  }

In cmdResearchCollect(), after parsing the query and before the tool detection block, add cache check:

  // Parse --no-cache flag
  const noCache = args.includes('--no-cache');

  // Cache check (skip if --no-cache, --quick, or rag_disabled)
  if (!noCache && !quick && ragEnabled && query) {
    const cache = getResearchCache();
    if (cache) {
      const cached = cache.getResearch(query);
      if (cached) {
        // Return cached result immediately — add cache_hit: true marker
        output({ ...cached, cache_hit: true }, { formatter: formatCollect, raw });
        return;
      }
    }
  }

At the end of cmdResearchCollect(), after building the result object but before calling output(), add cache write:

  // Cache write (skip if --no-cache)
  if (!noCache && result.source_count > 0) {
    const cache = getResearchCache();
    if (cache) {
      cache.setResearch(query, result);
    }
  }

Update formatCollect() to display cache status: if data.cache_hit is true, add a line: color.dim('(cached result — use --no-cache to refresh)')
