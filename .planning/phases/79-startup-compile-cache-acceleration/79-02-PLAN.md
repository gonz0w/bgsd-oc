<objective>
Fix the compile-cache feature so it actually works. The current implementation has a fundamental flaw: the CLI cannot add startup flags to its own running Node process. The solution requires a wrapper script that adds --experimental-code-cache before spawning Node.

Gap closure for Phase 79: RUNT-01 (faster startup) was NOT achieved because compile-cache flag was never applied.
</objective>

<context>
@.planning/phases/79-startup-compile-cache-acceleration/79-01-VERIFICATION.md

# Gap analysis from verification
- Gap 1: Compile-cache flag is never actually applied to Node process
- Gap 2: Benchmark shows -6% speedup (slower), not improvement
- Gap 3: RUNT-01 NOT ACHIEVED

# Root cause
The CLI cannot modify its own startup flags. getCompileCacheArgs() returns ['--experimental-code-cache'] but these args are discarded because the process has already started.

# Solution approach
Create a wrapper script (bin/bgsd) that:
1. Checks BGSD_COMPILE_CACHE env var
2. If enabled, spawns Node with --experimental-code-cache flag
3. If disabled or unsupported, spawns Node normally (current behavior)
4. Passes through all arguments to the actual CLI

This is the standard pattern used by tools like npx, yarn, etc.
</context>

   - If disabled or unsupported: spawn Node without the flag (current behavior)

2. The wrapper should:
   - Be a shell script (#!/bin/sh) for maximum compatibility
   - Check BGSD_COMPILE_CACHE env var
   - Detect Node version using --version flag
   - Use exec to replace the shell process with Node
   - Handle errors gracefully (if Node not found, show helpful message)

3. Make it executable: chmod +x bin/bgsd

Example structure:
```sh
#!/bin/sh
# Check if compile-cache should be enabled
if [ "${BGSD_COMPILE_CACHE:-0}" = "1" ]; then
  # Check Node version supports compile-cache
  NODE_VERSION=$(node --version | sed 's/v//' | cut -d. -f1)
  if [ "${NODE_VERSION:-0}" -ge 10 ]; then
    exec node --experimental-code-cache "$0-tools.cjs" "$@"
  fi
fi
# Fallback: run without compile-cache
exec node "$0-tools.cjs" "$@"
```

Actually, better approach - use the script location to find bgsd-tools.cjs:
```sh
#!/bin/sh
SCRIPT_DIR="$(cd "$(dirname "$0")" && pwd)"
CLI="$SCRIPT_DIR/bgsd-tools.cjs"

if [ "${BGSD_COMPILE_CACHE:-0}" = "1" ]; then
  NODE_VERSION=$(node --version | sed 's/v//' | cut -d. -f1)
  if [ "${NODE_VERSION:-0}" -ge 10 ]; then
    exec node --experimental-code-cache "$CLI" "$@"
  fi
fi
exec node "$CLI" "$@"
```
