Create `src/plugin/advisory-guardrails.js` with a factory function `createAdvisoryGuardrails(cwd, notifier, config)` that returns `{ onToolAfter, setBgsdCommandActive, clearBgsdCommandActive }`.

**Module structure:**

1. **Inline naming classifiers** â€” Copy NAMING_PATTERNS regex object and classifyName() from src/lib/conventions.js (5 patterns: camelCase, PascalCase, snake_case, kebab-case, UPPER_SNAKE_CASE). Plugin is ESM so cannot import CJS conventions.js.

2. **Factory function `createAdvisoryGuardrails(cwd, notifier, config)`:**
   - On init, load convention rules:
     - Primary: Read AGENTS.md from `cwd`, extract any naming convention mentions (regex for 'kebab-case', 'camelCase', etc.)
     - Fallback: Read `.planning/codebase/codebase-intel.json`, use `conventions.naming.dominant` if confidence >= 70%
     - Cache both reads (no re-reads per call)
   - On init, detect test command:
     - Read `package.json` from `cwd`, extract `scripts.test` (or `scripts.check`)
     - Detect source extensions from package.json (JS project â†’ `.js,.ts,.cjs,.mjs,.jsx,.tsx`)
     - Fall back to generic "run your tests" if no test script found
     - Cache result
   - Build static PLANNING_COMMANDS mapping table:
     ```
     'ROADMAP.md' â†’ ['/bgsd-add-phase', '/bgsd-remove-phase', '/bgsd-insert-phase']
     'STATE.md' â†’ ['/bgsd-progress', '/bgsd-execute-phase']
     'PLAN.md' â†’ ['/bgsd-plan-phase']
     'CONTEXT.md' â†’ ['/bgsd-discuss-phase']
     'RESEARCH.md' â†’ ['/bgsd-research-phase']
     'REQUIREMENTS.md' â†’ ['/bgsd-new-milestone']
     'config.json' â†’ ['/bgsd-settings']
     'SUMMARY.md' â†’ ['/bgsd-execute-phase']
     'INTENT.md' â†’ ['/bgsd-new-project', '/bgsd-new-milestone']
     ```
   - State variables:
     - `bgsdCommandActive` flag (boolean, default false) â€” set/cleared by index.js hooks
     - `warnCounts` Map (type â†’ count) for dedup threshold (first 3 individual, then summarize)
     - `testBatchTimer` and `testBatchFiles` array for debounced test suggestions
     - `testDebounceMs` from `config.advisory_guardrails?.test_debounce_ms || 500`

3. **`async onToolAfter(input)` method:**
   - Early return if `config.advisory_guardrails?.enabled === false`
   - Extract `tool` and `args` from input. Only process `write`, `edit`, `patch` tools
   - Extract `filePath` from `args?.filePath`. Early return if no filePath
   - Resolve filePath to absolute (join with cwd if relative)
   - Skip if filePath is in node_modules, tmp dirs, or other non-project paths

   **GARD-02 check (planning file protection) â€” runs first, takes priority:**
   - If filePath contains `.planning/`:
     - If `bgsdCommandActive` is true â†’ suppress (it's a workflow-driven write)
     - Otherwise: look up basename in PLANNING_COMMANDS table
     - If match: notify with severity='warning', type='advisory-planning', message naming the specific commands
     - If no match: generic "File in .planning/ was edited directly. bGSD workflows manage these files automatically."
     - Return early (don't also run GARD-01 on .planning/ files)

   **GARD-01 check (convention violations):**
   - Skip if no conventions were detected at init time
   - Extract basename without extension from filePath
   - Classify using classifyName()
   - If result is 'single-word' or 'mixed' â†’ skip (no convention to compare against)
   - If result doesn't match the project's dominant convention:
     - Apply dedup: increment warnCounts for type 'convention'. If count <= 3, send individual warning. If count > 3 and count is a multiple of 5 (or some threshold), send summary "...and N more convention warnings"
     - Notify: severity='warning', type='advisory-convention', verbose message: "File uses {detected} naming ({filename}). Project convention is {expected} (see AGENTS.md). Consider renaming to {suggested}."
     - Generate suggested name by converting filename to the expected convention (simple conversion: split on case boundaries, rejoin with convention separator)

   **GARD-03 check (test suggestions):**
   - Skip if filePath is a test file (contains `.test.`, `.spec.`, `__tests__/`, `test/`)
   - Check extension against detected source extensions
   - If source file: add to testBatchFiles, reset debounce timer
   - When timer fires (flushTestBatch):
     - Build suggestion message: single file â†’ "Modified {basename}. Consider running: {testCommand}"
     - Multiple files â†’ "Modified {count} source files. Consider running: {testCommand}"
     - Notify: severity='info', type='advisory-test', message as above

4. **`setBgsdCommandActive()` / `clearBgsdCommandActive()` methods:**
   - Simple flag toggle for the bgsdCommandActive boolean
   - Used by index.js command.execute.before / after hooks

5. **Helper functions (all internal):**
   - `loadConventionRules(cwd)` â€” reads AGENTS.md + codebase-intel.json, returns `{ dominant: string, confidence: number } | null`
   - `detectTestConfig(cwd)` â€” reads package.json, returns `{ command: string, sourceExts: Set<string> }`
   - `isTestFile(filePath)` â€” returns true if path indicates a test file
   - `toConvention(name, convention)` â€” converts a filename to the target naming convention (split on camelCase boundaries, PascalCase, snake_case delimiters, rejoin with convention separator)

**Constraints:**
- All file reads (AGENTS.md, package.json, codebase-intel.json) happen at init time, cached, NOT per tool call
- Use readFileSync (acceptable at init time, same as other plugin modules)
- Export only `createAdvisoryGuardrails` â€” all helpers are module-private
- Follow existing code style: JSDoc comments, similar structure to stuck-detector.js
- Keep bundle contribution under 5KB â€” no heavy logic, simple regex classifiers
