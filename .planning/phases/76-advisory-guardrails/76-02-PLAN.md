   ```javascript
     stuckDetector.trackToolCall(input);

   ```

5. **Extend commandEnrich hook** to set/clear bgsdCommandActive flag:
   In the existing `commandEnrich` safeHook callback, BEFORE the enrichCommand call, check if the command is a /bgsd-* command and set the flag:
   ```javascript
   const commandEnrich = safeHook('command.enrich', async (input, output) => {
     const cmdDir = directory || process.cwd();
     // Set guardrails flag for bGSD commands — suppresses planning file warnings
     if (input?.command && input.command.startsWith('bgsd-')) {
       guardrails.setBgsdCommandActive();
     }
     enrichCommand(input, output, cmdDir);
   });
   ```

   Note: The flag is cleared automatically after a timeout or by adding a command completion handler. Since `command.execute.before` fires at the start and there's no explicit `command.execute.after`, use a timeout-based clear (e.g., 30 seconds) inside the guardrails module itself, or clear on the next idle event. The simplest approach: in the `onToolAfter` method, if bgsdCommandActive has been set for more than 30s, auto-clear it. Alternatively, call `clearBgsdCommandActive()` in the idle handler.

6. **Clear guardrails flag on idle** — extend eventHandler:
   ```javascript
   const eventHandler = safeHook('event', async ({ event }) => {
     if (event.type === 'session.idle') {
       await idleValidator.onIdle();
       guardrails.clearBgsdCommandActive();
     }
     if (event.type === 'file.watcher.updated') {
       const { invalidateAll } = await import('./parsers/index.js');
       invalidateAll(projectDir);
     }
   });
   ```
