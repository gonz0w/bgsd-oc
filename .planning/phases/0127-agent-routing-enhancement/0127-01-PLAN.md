**In src/lib/decision-rules.js:**

3. Add three new pure decision functions following the exact contract `(state) => { value, confidence, rule_id, metadata? }`:

**resolveFileDiscoveryMode(state)**
- Inputs: `tool_availability` (object with boolean per tool), `scope` (string: 'single-file' | 'directory' | 'project-wide')
- Logic:
  - If `scope === 'single-file'`: return `{ value: 'node', confidence: 'HIGH', rule_id: 'file-discovery-mode' }` (no tool needed for single file)
  - If `tool_availability.fd === true` and scope is 'directory' or 'project-wide': return `{ value: 'fd', ... }`
  - Else: return `{ value: 'node', ... }` (Node.js fs fallback)
- Per CONTEXT.md: return just the tool name, hardcoded fallback chain (fd -> node), no config override

**resolveSearchMode(state)**
- Inputs: `tool_availability` (object), `needs_gitignore_respect` (boolean, default true)
- Logic:
  - If `tool_availability.ripgrep === true`: return `{ value: 'ripgrep', ... rule_id: 'search-mode' }` (ripgrep respects .gitignore by default)
  - If `tool_availability.fd === true` and `needs_gitignore_respect`: return `{ value: 'fd', ... }` (fd respects .gitignore)
  - Else: return `{ value: 'node', ... }` (Node.js fallback — does NOT respect .gitignore automatically)
- Hardcoded fallback chain: ripgrep -> fd -> node

**resolveJsonTransformMode(state)**
- Inputs: `tool_availability` (object), `json_complexity` (string: 'simple' | 'complex')
- Logic:
  - If `json_complexity === 'simple'`: return `{ value: 'javascript', ... rule_id: 'json-transform-mode' }` (no tool needed for simple parse/stringify)
  - If `tool_availability.jq === true` and `json_complexity === 'complex'`: return `{ value: 'jq', ... }`
  - Else: return `{ value: 'javascript', ... }` (JavaScript fallback)
- Hardcoded fallback chain: jq -> javascript (for complex); always javascript for simple

4. Register all three in `DECISION_REGISTRY` array with proper metadata:
   - Category: `'tool-routing'` (new category)
   - Each entry: id, name, category, description, inputs, outputs, confidence_range, resolve
   - Add a `// Phase 127: Tool routing decision functions` comment before the new entries

5. Export all three functions in the `module.exports` block under a `// Phase 127: Tool routing decision functions` comment.
